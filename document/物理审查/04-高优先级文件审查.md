# 物理框架审查第 3 轮：小型高优先级文件审查

本轮只读审查以下 3 个文件：

- `src/features/experiment/helpers.js`

## 文件逐项审查

### `src/features/experiment/constants.js`

审查结论：整体可作为后续数值模型基线。物理常量、材料逸出功、滤光片波长/频率、电压量程和电流量程集中定义，符合 K001-K005、K006、K012 对“统一物理口径”的要求。

关键代码证据：

- `PLANCK_EV = 4.135667696e-15`、`PLANCK_JS = 6.62607015e-34`、`SPEED_OF_LIGHT = 2.99792458e8`、`ELEMENTARY_CHARGE = 1.602176634e-19`：`src/features/experiment/constants.js:3`
- `MATERIALS` 使用 eV 数值保存材料逸出功：`src/features/experiment/constants.js:8`
- `FILTERS` 保存滤光片波长和频率，其中频率按 `10^14 Hz` 量级保存：`src/features/experiment/constants.js:16`
- 截止电压模式电压量程为 `-2-2`，伏安模式为 `-2-30`：`src/features/experiment/constants.js:68`

合理简化：

- 材料逸出功采用固定预设值，符合 K004/K015 的教学模型边界，但不能代表真实材料表面状态、污染层或接触电势差。
- 滤光片只记录中心波长/频率，没有带宽、透过率或汞灯谱线强度，符合 K014 的“可审查换算，不宜严判带宽”的边界。
- `PHOTOCURRENT_SATURATION_CURRENT_MIN_AMP` 和 `MAX` 给出教学级饱和电流范围，是否导致 0 光强仍有光电流，需要后续在 `experiment.js` 中对照计算逻辑确认。

### `src/features/experiment/helpers.js`

审查结论：默认状态、量程、电压格式、线性回归和报告摘要逻辑基本符合 K003、K005、K012、K013；但调零偏差输出存在明确的安培级精度丢失问题。

关键代码证据：

- 默认状态设置频率为 `FILTERS[0].frequency * 1e14`，材料为 `cesium`，电压为 `-1.998`：`src/features/experiment/helpers.js:115`
- 电压按测量模式量程夹取：`src/features/experiment/helpers.js:163`
- 电流显示按 `current / currentRange` 缩放，调用端传入 `currentPhotocurrentAmp`：`src/features/experiment/helpers.js:215`，`src/stores/experiment.js:331`
- 线性回归使用 `frequency` 为 x、`cutoffVoltage` 为 y：`src/features/experiment/helpers.js:315`
- 普朗克常量估计使用 `regression.slope * ELEMENTARY_CHARGE`：`src/features/experiment/helpers.js:399`
- 截距绝对值用于估算逸出功，阈频为 `workFunction / PLANCK_EV`：`src/features/experiment/helpers.js:402`

合理简化：

- `buildAnalysisSummary()` 使用正值 `cutoffVoltage` 拟合 `U_c - nu`，与 K005/K013 一致。后续仍需在 `experiment.js` 中确认记录值始终是正的截止电压，而不是带符号的反向外加电压。
- 线性回归没有直接处理外推法或曲率法，只提供基础线性拟合，符合当前教学仿真的最小模型；若要覆盖 K013 中更多数据处理方法，需要后续作为增强项。
- 默认电压接近 `-2 V`，更像截止电压测试初始状态；是否影响新用户初始数据记录，需要在 store 的记录校验中确认。

### `src/features/photoelectric-animation/physics.js`

审查结论：该文件是动画单位下的运动工具，不是 SI 单位真实电子轨迹求解器。它清楚使用 `forceMagnitudeProxy`、`animationAccelerationPxPerS2` 等代理量，整体符合 K010/K015 的教学动画简化边界。

关键代码证据：

- 正电压映射为电子朝收集极方向加速，负电压映射为反向减速：`src/features/photoelectric-animation/physics.js:24`
- 场强比例按电压绝对值与参考电压归一：`src/features/photoelectric-animation/physics.js:36`
- 加速度使用动画缩放常量 `FIELD_TO_ANIMATION_ACCELERATION`：`src/features/photoelectric-animation/physics.js:59`
- 初速度按 `sqrt(kineticEnergyEv)` 映射到动画速度：`src/features/photoelectric-animation/physics.js:68`
- 电子飞行支持收集、返回阴极、离开场区和失败状态：`src/features/photoelectric-animation/physics.js:105`
- 单元测试明确验证了正电压加速、负电压减速、截止强度反向场返回、正向场收集和伏安轨迹离开场区继续飞行：`src/features/photoelectric-animation/physics.spec.js:15`

合理简化：

- `EV_TO_ANIMATION_SPEED` 和 `FIELD_TO_ANIMATION_ACCELERATION` 是动画比例尺，不代表真实速度或真实加速度，符合 K015。
- `createElectronFlightState()` 将初始 x 速度限制为非负：`src/features/photoelectric-animation/physics.js:98`。这不支持从基础函数直接生成反向初速度电子，但在教学动画里可理解为“从阴极向阳极方向逸出”的简化；是否缺少角分布，需要在 `mapping.js` 和 `engine.js` 中继续确认。
- 电场方向使用项目约定：正电压推动电子向收集极，负电压减速/返回。后续审查应确认所有调用方使用同一符号约定。

## 需要后续对照大文件确认的点

1. `PHOTOCURRENT_SATURATION_CURRENT_MIN_AMP` 是否会导致光强为 0 或低于阈频时仍生成不合理阴极光电流，需要在 `experiment.js` 中确认。
2. `cutoffVoltage` 记录值是否始终是正值截止电压；若记录的是反向外加电压，`buildRegression()` 和 `dataDiagnostic.js` 的趋势判断会出现符号错误。
3. `mapping.js` 是否始终把光强映射到光子通量/发射率/饱和电流，而不是最大初动能。
4. `physics.js` 的正负电压约定是否与 `mapping.js`、`voltAmpTrajectory.js` 和 `engine.js` 完全一致。
5. `evaluateZeroReference()` 的偏差输出是否被 UI 或报告展示；若展示，需要优先修复精度格式。
