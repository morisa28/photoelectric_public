# 物理框架审查第 6 轮：综合审查报告

## 审查结论概览

当前项目在光电效应核心关系上已有较完整的教学仿真框架：

- 频率/波长、材料逸出功、最大初动能、截止电压之间的主公式基本正确。
- 光强主要影响光子事件密度、发射率和饱和电流，没有发现把光强直接映射为单电子最大初动能的核心错误。
- 伏安模式使用角度、速度和位置采样估计收集率，符合概率化动画表达。
- 微观动画文案基本能区分频率、光强、电压、暗电流、本底电流和阳极污染寄生项。

主要风险集中在：

1. 正常阴极光电流在反向超过截止电压后被 `tanh` 模型算成负值。
2. 截止电压动画门控依赖真实电流输入；纯教学脚本缺少电流输入时可能在“截止电压”步骤仍显示少量收集。
3. 诊断和 AI 摘要没有稳定区分截止电压记录与伏安扫描记录。
4. 调零偏差输出和若干字段命名存在单位/精度误导风险。

## 优先级说明

- P1：会造成核心教学误导或核心数值模型错误，但仍可通过局部模型修正解决。
- P2：会影响特定模式、数据解释或用户理解，需要修正，但不是全局公式崩坏。
- P3：命名、健壮性、文案或维护风险。

本轮未标 P0。原因是频率-截止电压、光强-电流、伏安连续曲线、概率化动画等主框架大体成立；发现的问题没有达到“整个物理框架不可用”的程度。

## P1 问题

### P1-001 正常阴极光电流在反向过截止后变成负值

- 涉及文件：
  - `src/stores/experiment.js`
  - `src/components/PrincipleAnimationSection.vue`
- 代码证据：
  - `generatedPhotocurrentAmp()` 使用 `saturationCurrent * Math.tanh((voltage + cutoff) / 0.45)`：`src/stores/experiment.js:221`
  - 原理动画 `trueCathodePhotocurrentAmp()` 使用同类公式：`src/components/PrincipleAnimationSection.vue:381`
- 理论依据：
  - K003 最大初动能与遏止/截止电压关系。
  - K007 光电流-电压曲线分区与连续性。
  - K009 反向电压、截止电流与弱信号拖尾。
  - K011 本底电流、暗电流和阳极寄生项。
- 问题说明：
  - 正常阴极光电流应随反向电压增强而下降到接近 0。
  - 负向电流应由阳极光电流、暗电流、本底电流或其他寄生项解释。
  - 当前 `tanh` 模型在 `voltage < -cutoff` 时会产生负的“正常阴极光电流”，混淆了正常光电流和寄生/误差电流。
- 影响范围：
  - 2D/真实实验 store 电流读数。
  - 伏安曲线反向区。
  - 3D 微观动画适配层读取的 `trueCathodePhotocurrentAmp`。
  - 原理动画区的“真实阴极光电流”展示。
  - 数据记录中的 `current` 字段。
- 建议修改方向：
  - 将正常阴极光电流模型下限限制为 0。
  - 单独保留寄生阳极电流、暗电流、本底电流作为负向或残余项。
  - 用平滑函数表达截止附近拖尾，例如对正常阴极电流使用非负 sigmoid/tanh 变体，而不是允许符号翻转。
- 证据等级：A。

## P2 问题

### P2-001 教学脚本截止步骤缺少真实电流门控

- 涉及文件：
  - `src/features/photoelectric-animation/tutorialScript.js`
  - `src/features/photoelectric-animation/mapping.js`
- 代码证据：
  - 第 19 步“截止电压”只给出 `voltage: -0.95` 等预设，没有传入真实/测量电流：`src/features/photoelectric-animation/tutorialScript.js:27`
  - `currentGateState` 依赖传入电流水平：`src/features/photoelectric-animation/mapping.js:366`
  - `resolveCutoffModeCollection()` 在反向状态有 `0.02` 收集率下限：`src/features/photoelectric-animation/mapping.js:219`
- 理论依据：K003、K009、K015。
- 问题说明：
  - 实时原理动画和真实实验适配层会传入真实阴极电流，因此能触发 cutoff gate。
  - 教学脚本直接调用映射函数，没有电流门控，导致“截止电压”步骤可能仍表现为部分收集。
- 建议修改方向：
  - 教学脚本 cutoff 步骤显式传入接近零的 `trueCathodePhotocurrentAmp` 和 `measuredPhotocurrentAmp`。
  - 或在 `mapping.js` 中增加基于 `abs(voltage) >= kineticEnergyEv` 的备用 cutoff 判定。
- 证据等级：B。

### P2-002 无电流输入时反向收集率保留 2% 下限

- 涉及文件：`src/features/photoelectric-animation/mapping.js`
- 代码证据：反向状态中 `resolveCutoffModeCollection()` 返回 `clamp(..., 0.02, 0.72)`：`src/features/photoelectric-animation/mapping.js:219`
- 理论依据：K003、K007、K009。
- 问题说明：
  - 该下限可用于表达截止附近拖尾，但当反向电压已经超过最大初动能且没有寄生项时，正常阴极光电流不应保留固定正常收集率。
- 建议修改方向：
  - 将拖尾与寄生/暗电流区分开。
  - 正常阴极收集率应允许在超过截止条件后降为 0。
- 证据等级：B。

### P2-003 数据诊断未过滤伏安扫描记录

- 涉及文件：
  - `src/services/dataDiagnostic.js`
  - `src/components/AIAssistant.vue`
- 代码证据：
  - 诊断直接使用全部记录的 `cutoffVoltage`：`src/services/dataDiagnostic.js:24`、`src/services/dataDiagnostic.js:57`
  - AI 摘要对所有记录调用 `r.cutoffVoltage.toFixed(3)`：`src/components/AIAssistant.vue:182`
- 理论依据：K013。
- 问题说明：
  - 自动伏安扫描记录的 `cutoffVoltage` 为 `null`。
  - 诊断服务会把 `null` 当作 0 参与趋势和回归，AI 摘要还可能直接报错。
- 建议修改方向：
  - 所有 `U_c - nu` 诊断只使用 `measurementMode === 'cutoff'` 且 `cutoffVoltage` 有限的记录。
  - AI 摘要按截止电压记录和伏安扫描记录分组描述。
- 证据等级：B。

### P2-004 调零偏差输出丢失安培级精度

- 涉及文件：`src/features/experiment/helpers.js`
- 代码证据：`evaluateZeroReference()` 返回 `Number(deviation.toFixed(6))`，安培级 `1e-13` 会被格式化为 0：`src/features/experiment/helpers.js:298`
- 理论依据：K012。
- 问题说明：
  - 判定逻辑在四舍五入前完成，因此不直接影响 `ok`。
  - 但展示/诊断中的偏差值会误导用户以为偏差为 0。
- 建议修改方向：
  - 返回原始 `deviation` 数值。
  - 展示层使用 `toExponential(6)`。
- 证据等级：B。

## 建议修改顺序

1. 先修复 P1-001：正常阴极光电流不得为负。
2. 同步修复原理动画中的同类 `trueCathodePhotocurrentAmp()`。
3. 调整 cutoff 门控：让正常收集率在超过截止条件后可归零，寄生/暗电流单独显示。
4. 修复诊断和 AI 摘要的数据过滤，避免伏安扫描记录污染 `U_c - nu` 分析。
5. 修复调零偏差展示精度。
6. 做命名和常量复用清理。

## 建议验证项

后续若进入代码修改，应至少补充以下测试：

1. 固定频率/材料下，`voltage <= -currentCutoffVoltage` 时正常阴极光电流为 0 或接近 0，不能为负。
2. 阳极寄生电流开启时，负向测量电流来自寄生项而不是正常阴极项。
3. 教学脚本第 19 步 cutoff 状态收集率为 0 或接近 0。
4. 伏安扫描记录不参与 `U_c - nu` 拟合和 AI 截止电压摘要。
5. 调零偏差在 `1e-13 A` 量级显示为指数格式而不是 0。
