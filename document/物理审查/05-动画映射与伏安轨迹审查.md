## 文件逐项审查

### `src/features/photoelectric-animation/mapping.js`

审查结论：主物理关系方向基本正确。频率进入光子能量、最大初动能和阈频；光强主要进入光子事件率、发射率和电流水平；电压进入电场、加速度、收集率和伏安轨迹估计。整体符合 K001-K011、K015。

关键代码证据：

- `photonEnergyEv = PLANCK_EV * frequency`，`kineticEnergyEv = photonEnergyEv - workFunctionEv`，`thresholdFrequency = workFunctionEv / PLANCK_EV`：`src/features/photoelectric-animation/mapping.js:299`
- `hasIncidentLight` 要求灯亮、光路打开且光强大于 0：`src/features/photoelectric-animation/mapping.js:307`
- `canEmit` 默认要求有入射光且 `kineticEnergyEv > 0`：`src/features/photoelectric-animation/mapping.js:308`
- 光强进入 `normalizedIntensity`、`photonRate`、`escapeProbability`、`modeledCurrentLevel`：`src/features/photoelectric-animation/mapping.js:304`、`src/features/photoelectric-animation/mapping.js:391`、`src/features/photoelectric-animation/mapping.js:419`、`src/features/photoelectric-animation/mapping.js:430`
- 电压进入 `calculateElectricField()` 和 `estimateVoltAmpCollectionRatio()`：`src/features/photoelectric-animation/mapping.js:381`、`src/features/photoelectric-animation/mapping.js:395`
- 阳极污染电流仅在反向电压且光照有效时生成负向寄生项：`src/features/photoelectric-animation/mapping.js:318`

合理简化：

- `cutoffVoltage` 输出字段实际等于 `kineticEnergyEv`：`src/features/photoelectric-animation/mapping.js:468`。由于 `1 eV` 最大初动能对应 `1 V` 遏止电压数值，这在教学模型中数值可等价，但字段名有单位语义风险。
- `lowEnergyBackgroundRate` 在有入射光时生成，即使低于阈频也可能有低能背景事件：`src/features/photoelectric-animation/mapping.js:420`。如果引擎把这些事件表现为失败/回落，可作为教学演示；如果表现为正常收集，会与 K002/K004 冲突。
- `mapping.js` 自行定义 `PLANCK_EV`、`PHOTOCURRENT_LEVEL_REFERENCE_AMP` 等常量，而不是复用 `experiment/constants.js`，当前数值一致，但存在后续漂移风险。

### `src/features/photoelectric-animation/voltAmpTrajectory.js`

审查结论：伏安轨迹采样采用角度分布、速度因子、源位置偏移和飞行时间判定，符合 K007/K009/K010 对“连续伏安曲线”和“概率化收集”的教学表达要求。

关键代码证据：

- 角度采样覆盖 `-88` 到 `88` 度：`src/features/photoelectric-animation/voltAmpTrajectory.js:11`
- 速度因子和源位置偏移用于构造分布：`src/features/photoelectric-animation/voltAmpTrajectory.js:35`
- 电压正负决定加速度方向，正电压加速、负电压减速：`src/features/photoelectric-animation/voltAmpTrajectory.js:95`
- 到达时间用匀加速位移方程求根：`src/features/photoelectric-animation/voltAmpTrajectory.js:54`
- 是否收集由是否到达阳极窗口、是否离开场区共同决定：`src/features/photoelectric-animation/voltAmpTrajectory.js:198`
- 收集率由收集样本数除以总样本数给出：`src/features/photoelectric-animation/voltAmpTrajectory.js:227`

合理简化：

- 采样分布是离散角度/速度/位置网格，不是真实电子角分布或能量分布。
- 伏安模式使用独立动画尺度 `VOLT_AMP_INITIAL_SPEED_SCALE` 和 `VOLT_AMP_ACCELERATION_SCALE`，是为了前端展示连续轨迹，不能作为真实飞行时间。
- `DEFAULT_REFERENCE_VOLTAGE = 30` 适配伏安模式 `-2-30` 的电压范围，符合项目量程设置。

### `src/features/photoelectric-animation/tutorialScript.js`

审查结论：教学脚本提供 20 个演示步骤，能覆盖频率、光强、逸出功、电场、截止电压等教学点。但脚本直接调用 `resolvePhotoelectricAnimationState()` 时没有传入真实阴极电流或测量电流，导致截止附近的电流门控不能像实时原理动画那样关闭。

关键代码证据：

- 第 19 步标题为“截止电压”，输入为 `voltage: -0.95`、`frequency: 7.85e14`、`workFunction: 2.3`：`src/features/photoelectric-animation/tutorialScript.js:27`
- 教学脚本只传入 `animationInput`、`focusTarget`、`presentationMode`，未传入 `trueCathodePhotocurrentAmp` 或 `measuredPhotocurrentAmp`：`src/features/photoelectric-animation/tutorialScript.js:46`
- `resolveVoltageState()` 只有在 `currentGateState === 'cutoff'` 时才返回 `cutoff-near`：`src/features/photoelectric-animation/mapping.js:120`
- `currentGateState` 又依赖传入的真实/测量电流水平：`src/features/photoelectric-animation/mapping.js:366`

## 需要后续对照大文件确认的点

1. `engine.js` 中 `energyKind: 'low'` 的低能背景事件是否会被收集并计入普通电流。
2. `engine.js` 的统计窗口是否使用 `collectorSuccessRatio`，以及 cutoff 状态是否可靠地停止正常电子收集。
3. `experiment.js` 中真实阴极电流在截止附近是否会变为足够低，从而为实时动画提供正确 `currentGateState`。
4. `PhotoelectricAnimationPanel.vue` 是否在界面文案中把 `cutoffVoltage`、`kineticEnergyEv`、`Kmax` 的单位讲清楚。
