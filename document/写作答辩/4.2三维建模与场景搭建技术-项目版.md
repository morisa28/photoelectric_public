# 4.2 三维建模与场景搭建技术

本项目的三维建模与场景搭建围绕“光电效应实验模拟仿真平台”的教学目标展开。与传统只展示静态模型的三维页面不同，本项目将三维实验装置、实时实验参数、控制面板交互和教学辅助可视化整合到同一网页运行环境中，使用户能够在 2D 实验台与 3D 实验场景之间切换，并保持实验状态一致。

当前项目采用 Vue 3 作为前端框架，使用 Pinia 维护统一实验状态，三维场景由 Spline 制作并导出为 `scene.splinecode`，再通过项目内置的 `@splinetool/runtime` 在 `/3d-view` 页面加载运行。该方案既保留了三维实验台的沉浸感，又避免在网页端手动维护大量底层 Three.js 建模逻辑，便于后续持续迭代、视觉检查和交互修复。

[图占位：图7 三维场景开发工具与工程入口图。应展示 Spline 场景编辑界面、项目中的 `resource/scene.splinecode`、`src/components/SplineScene.vue`、`src/views/ThreeDView.vue` 等关键入口。]

## 4.2.1 实验场景整体布局设计

### 1. 场景规划与布局

本项目以真实光电效应实验仪器的操作关系为基础，将三维场景划分为实验台主体、光源系统、滤光片系统、光电管、控制面板、显示屏和连接线缆等区域。场景中央保留主要实验仪器，使用户进入 3D 页面后能够直接观察到实验装置全貌；控制按钮和显示区域集中布置在仪器正面，便于完成电压调节、模式切换、量程选择、调零确认和数据记录等操作。

场景布局遵循“实验装置居中、控制区前置、状态反馈可见”的原则。光源、滤光片和光电管按光路方向排列，体现光从汞灯出射、经过滤光片筛选、最终进入光电管的实验路径。控制面板则布置在用户视角容易触达的位置，减少三维操作时的寻找成本。

[图占位：图8 光电效应实验整体场景搭建图。应展示 3D 实验台全貌，包含光源、滤光片、光电管、控制面板、双显示屏和实验室环境背景。]

### 2. 页面承载与观察视角

三维场景由独立页面 `/3d-view` 承载，页面文件为 `src/views/ThreeDView.vue`。该页面负责挂载 `SplineScene.vue`，并提供返回 2D 实验台、记录当前数据、查看数据、打开诊断模式和打开微观原理演示小窗等操作入口。

场景运行层由 `src/components/SplineScene.vue` 负责。组件通过 `Application` 创建 Spline 运行实例，并加载 `resource/scene.splinecode`。页面支持 `diagnostic`、`wait`、`zoom`、`background` 等查询参数，便于在开发和验收阶段固定视角、等待时序和背景颜色，从而稳定生成视觉截图和运行时诊断报告。

[图占位：图9 3D 页面运行效果图。应展示 `/3d-view` 页面、顶部工具栏、主三维实验台以及诊断模式下的场景状态信息。]

### 3. 场景细节与优化

在场景细节处理上，项目重点强化了实验仪器的可识别性和可操作性。双显示屏、按钮、旋钮、滤光片、光电管和线缆都被设计为可被运行时定位或交互的对象。对象命名在代码中具有明确映射，例如 `left_plane`、`right_plane` 用于电流和电压屏幕贴图，`Phototube_Box` 用于光电管拖拽，`Filter_Rotator` 用于滤光片旋转，`Knob_CurrentRange` 和 `Knob_ZeroAdjust` 用于量程与调零旋钮状态同步。

为了适应网页端运行，场景没有在前端继续拼装多个分散 GLB 模型，而是将最终三维结果收束为单一 Spline 场景资源。早期项目中曾保留过 Three.js、本地 GLB/HDR 资源和模型桥接链路，后续版本已将这部分清理，减少运行路径分叉，使 3D 页面维护重点集中到 Spline 场景加载、对象查找、材质贴图刷新和交互事件映射上。

## 4.2.2 光电效应实验仪器建模

### 1. 仪器结构分析与分区

光电效应实验仪器的三维建模围绕实验流程进行功能分区，主要包括：

1. 光源系统：用于表示汞灯和入射光源，是光电效应发生的起点。
2. 滤光片系统：用于体现不同波长光的选择过程，并与实验状态中的滤光片波长同步。
3. 光电管区域：用于表示光电子发射和收集的实验主体，并支持距离调节。
4. 控制面板：承载电源、工作模式、测量模式、电压调节、量程选择、调零确认等交互。
5. 双显示屏：分别显示实时光电流与电压信息。
6. 线缆与接线区域：用于表达仪器连接状态，并与后端实验状态联动。

这种分区方式使模型不仅具有展示作用，还能与实验业务状态建立稳定对应关系。用户在 3D 页面中看到的按钮、旋钮、屏幕和光电管位置变化，都不是孤立动画，而是由统一实验状态驱动。

[图占位：图10 光电效应实验仪器结构分区图。应标注光源系统、滤光片、光电管、控制面板、双显示屏、接线柱和线缆位置。]

### 2. 各部件建模与对象命名

控制面板部分是本项目三维建模与交互结合最密切的区域。面板上的按钮和旋钮在视觉上需要与真实实验仪器接近，同时还要具备运行时可定位的对象名称。项目在 `SplineScene.vue` 中维护了按钮与操作的映射关系，例如：

- `Btn_Left`、`Btn_Right` 用于调节电压精度。
- `Btn_Up`、`Btn_Down` 用于增加或降低电压。
- `Btn_AutoManual` 用于切换自动/手动工作模式。
- `Btn_Query001`、`Btn_VoltAmpTest` 用于切换测量模式。
- `Btn_Power` 用于切换仪器电源。
- `Btn_ZeroConfirm` 用于调零确认。
- `Line_current_input` 用于切换后部线缆连接状态。

旋钮与可拖拽对象同样通过名称绑定运行时逻辑。量程旋钮、调零旋钮、滤光片和光电管分别对应不同的拖拽或步进处理逻辑，交互结果统一回写 Pinia store，避免 3D 场景内部形成第二套实验状态。

[图占位：图11 控制面板与交互对象命名图。应展示按钮、旋钮、滤光片、光电管等对象名称与功能之间的对应关系。]

### 3. 材质与显示效果处理

三维仪器材质处理重点服务于“功能可读性”。双显示屏使用深色底面配合高亮数字，按钮、指示灯和线缆通过颜色差异区分功能状态。为了使显示内容能够随实验参数实时变化，项目没有直接使用 Spline 内部固定文本，而是在前端创建 Canvas 画布，将电流、电压和指示灯状态绘制成图片，再写入 Spline 材质层。

具体实现中，`left_plane` 用作电流显示屏，`right_plane` 用作电压显示屏。`SplineScene.vue` 为两个屏幕创建 1024x512 的离屏 Canvas，并根据当前实验状态绘制标签、单位、数字、闪烁位和屏幕光效。绘制完成后，通过 Spline runtime 的共享图片资源和材质 layer 更新机制，将 Canvas 数据作为纹理贴回对应模型表面。

[图占位：图12 双屏材质贴图同步示意图。应展示 Pinia 实验状态到 Canvas 绘制，再到 `left_plane`、`right_plane` 材质纹理更新的流程。]

## 4.2.3 模型导出与集成准备

### 1. 模型资源检查与运行优化

在模型集成前，需要保证三维对象具有明确命名、稳定层级和可被运行时查找的关键节点。对于本项目而言，模型检查不只关注视觉完整性，还关注以下内容：

1. 关键对象是否可通过名称查找，例如屏幕、按钮、旋钮、滤光片和光电管。
2. 可交互区域是否具备足够清晰的命中范围。
3. 显示屏材质是否允许运行时插入或更新纹理层。
4. 场景相机、灯光和背景是否适合网页端展示。
5. 场景资源体积是否适合通过 Vite 打包并在浏览器中加载。

项目当前保留的核心三维资源为 `resource/scene.splinecode`。该文件作为场景整体导出结果，由 Vite 通过 `?url` 方式处理，并在运行时由 Spline runtime 加载。相比多模型散装加载，这种方式降低了资源路径管理复杂度，也减少了模型节点版本不一致导致的交互失效。

### 2. 场景导出与前端集成

Spline 场景导出后放置在项目 `resource` 目录中。前端组件通过如下链路完成加载：

```text
router `/3d-view`
-> `src/views/ThreeDView.vue`
-> `src/components/SplineScene.vue`
-> `resource/scene.splinecode`
-> vendored `@splinetool/runtime`
-> WebGL canvas
```

`SplineScene.vue` 在组件挂载时创建 Spline `Application`，随后调用 `load(sceneUrl)` 加载场景。场景加载完成后，组件会同步旋钮状态、滤光片状态、线缆状态、光电管位置、显示屏纹理和诊断信息，并启动线缆渲染循环。

[图占位：图13 模型导出与网页集成流程图。应展示 Spline 导出 `scene.splinecode`、Vite 资源处理、`SplineScene.vue` 加载、WebGL canvas 呈现的完整链路。]

### 3. 版本收口与历史经验

项目早期开发中曾使用 Three.js 场景、GLB 模型、HDR 环境贴图和自定义相机控制模块来搭建 3D 实验台。这种方式灵活度高，但会使模型装配、交互命中、材质更新、相机控制和业务状态同步全部集中在前端代码中，维护成本较高。

后续迭代中，项目将 3D 场景主链路收口为 Spline 场景资源和运行时控制层：模型细节主要在 Spline 中维护，网页端只负责加载、状态同步、材质贴图刷新、交互事件映射和自动化诊断。这一调整使模型文件、页面组件和业务 store 的边界更清晰，也便于在不同阶段持续修复 3D 视觉和交互问题。

## 4.2.4 场景运行与交互集成

### 1. 统一实验状态驱动

本项目的 2D 页面与 3D 页面共用同一个 `experiment` store。电压、频率、光强、滤光片波长、光电管距离、仪器电源、工作模式、测量模式、量程、调零状态和数据记录等内容都以 Pinia store 为事实来源。3D 场景中的交互只负责把用户操作转换为 store action，显示层再根据 store 变化刷新。

这种状态流向可以概括为：

```text
3D 按钮/旋钮/拖拽操作
-> `SplineScene.vue` 命中判断与动作映射
-> Pinia experiment store
-> 计算光电流、电压、截止电压、记录状态
-> 2D 页面、3D 双屏、数据弹窗、微观小窗同步刷新
```

[图占位：图14 2D 与 3D 状态同步结构图。应展示 2D 实验台、3D 场景、Pinia store、数据图表和报告模块之间的状态流向。]

### 2. 控制面板交互实现

3D 控制面板的交互包含按钮点击和拖拽两类。按钮点击通过射线命中和屏幕投影兜底热区识别，最终映射到 `BUTTON_ACTIONS` 中的业务动作。拖拽交互主要用于量程旋钮、调零旋钮、滤光片和光电管距离调节。

其中，光电管拖拽已统一收口到 `Phototube_Box` 的射线命中路径，拖动结果按 `PHOTOCELL_DISTANCE_STEP` 写回光电管距离。滤光片旋转会更新滤光片波长，量程旋钮会同步当前电流量程，调零旋钮则影响调零值。所有交互都以 `source: '3d'` 标记写回 store，便于后续区分 2D 与 3D 操作来源。

[图占位：图15 3D 控制面板交互示意图。应展示按钮点击、旋钮拖拽、滤光片旋转和光电管拖动的操作区域。]

### 3. 数据记录与教学小窗集成

3D 页面顶部工具栏提供“记录当前数据”和“查看数据”功能。记录操作会调用实验 store 中的数据记录逻辑；查看数据则打开页面级数据弹窗，展示当前实验曲线和分析结果。这样用户在 3D 场景中完成操作后，无需回到 2D 页面即可查看实验数据。

此外，项目在 3D 页面中加入了“微观原理演示小窗”。该小窗不是 Spline 场景内部模型，而是与 `SplineScene` 同级的页面浮层。它读取主实验参数，通过 Vue 组件和 Canvas 2D 动画展示光子入射、能量吸收、阈频判断、电子逸出和电场收集等微观过程。该设计既增强了教学解释能力，又避免把教学动画逻辑塞入三维场景运行时。

[图占位：图16 微观原理演示小窗效果图。应展示光子、电子、金属表面、收集极、电场方向提示和讲解模式步骤。]

## 4.2.5 技术要点与经验总结

### 1. 场景建模服务于教学操作

三维场景建模不仅要追求外观还原，还要服务于实验操作路径。光源、滤光片、光电管、按钮、旋钮、显示屏和线缆都需要在视觉上明确可识别，并在运行时具备稳定名称和交互映射。只有模型结构与业务状态形成一一对应，用户操作才不会停留在表面点击，而能真正驱动实验流程。

### 2. 显示内容应由实验状态生成

项目实践表明，电流、电压和指示灯这类实时变化内容不适合固化在 3D 模型文本中。通过 Canvas 动态绘制并写入材质纹理，可以让三维仪器显示结果与物理计算、数据记录和 2D 页面保持一致，也便于后续调整字体、亮度、闪烁和量程提示效果。

### 3. 交互逻辑应避免分散状态源

3D 场景中的按钮、旋钮和拖拽操作都统一回写 Pinia store，而不是在 Spline 对象上保存独立业务状态。这样可以保证 2D、3D、数据图表、实验报告和微观小窗共享同一套实验事实，避免出现页面之间参数不一致的问题。

### 4. 自动化诊断保障持续迭代

3D 场景视觉问题通常难以仅靠代码阅读发现。本项目补充了 `npm run vis:check`、`npm run scene:report`、`npm run scene:strings` 和 `npm run scene:snap` 等脚本，用于生成固定视口截图、运行时对象报告和 Spline 场景字符串分析结果。后续涉及场景亮度、遮挡、按钮命中或屏幕对位问题时，可以先通过自动化报告定位，再进行针对性修改。

[图占位：图17 三维场景自动化诊断结果图。应展示固定截图矩阵、运行时对象统计、屏幕绑定状态和交互对象命中信息。]

综上，本项目的三维建模与场景搭建已经从早期的本地 Three.js 模型拼装，演进为以 Spline 场景资源为主体、Vue 页面承载、Pinia 状态驱动、Canvas 纹理同步和自动化视觉诊断协同工作的网页端 3D 实验方案。该方案既满足光电效应实验的沉浸式展示需求，也为后续扩展更多教学辅助功能和交互细节提供了稳定基础。
