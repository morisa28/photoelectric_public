## 总体答辩口径

我们平台的定位不是替代真实精密光电效应仪器，而是面向大学物理实验教学的虚拟仿真平台。核心目标是把光电效应中“频率决定单个光电子最大初动能、光强决定单位时间光电子数量和饱和光电流、反向电压决定电子收集情况、截止电压与频率线性相关”这些规律，通过实验操作、数据记录、曲线拟合、2D/3D 场景和微观动画串成一个完整学习闭环。

当前运行版的核心物理计算在前端 Vue + Pinia 状态模型中完成，AI 助手只做解释、异常提示和报告辅助，不直接决定物理结论。3D 主链路是 Spline 场景 + Vue 页面 + Pinia 统一状态源。平台已经实现了截止电压计算、简化 I-U 曲线、调零与记录流程、自动扫描、数据拟合、实验报告、知识图谱、AI 辅助问答、3D 场景操作和微观动画小窗；尚未严格实现真实仪器标定、滤光片带宽、接触电势差、空间电荷效应、完整电子初动能分布和费米-狄拉克数值模型。

## 一、物理核心问题

### 1. 你们平台中光电效应的底层物理模型具体是什么？是直接使用爱因斯坦光电方程，还是还加入了伏安特性曲线模型？

底层首先使用爱因斯坦光电方程：

```text
E = hν
hν = W0 + Ekmax
eUc = Ekmax
Uc = (h/e)ν - W0/e
```

截止电压、阈频和普朗克常量拟合都基于这个模型。

同时，平台还加入了一个教学化的 I-U 伏安响应模型。当前运行版不是严格由真实仪器数据拟合得到，也不是完整费米-狄拉克分布积分，而是用平滑函数模拟从反向遏止区到正向饱和区的连续变化，并叠加少量测量噪声和仪器偏置。也就是说：截止电压主模型是爱因斯坦方程，伏安曲线是教学仿真的简化平滑模型。

### 2. 你们在程序里是怎样由频率计算截止电压的？单位换算是怎么处理的？

程序中频率单位是 Hz，普朗克常量使用 `4.135667696e-15 eV·s`。这样 `hν` 直接得到电子伏特 eV 单位的光子能量。再减去材料逸出功 `W0(eV)`：

```text
Ekmax(eV) = h(eV·s)ν(Hz) - W0(eV)
```

对电子而言，最大动能的 eV 数值与对应遏止电压的 V 数值相同，所以：

```text
Uc(V) = hν - W0
```

如果计算结果小于 0，平台按 0 处理，表示低于截止频率，不产生可测光电流。

### 3. 你们如何区分“遏止电压”“截止电压”和“反向电压”？界面和数据处理里有没有混用？

我们这样区分：

- 反向电压：外加电压方向，与光电子运动方向相反，用来减速或阻止光电子到达阳极。
- 截止电压/遏止电压：刚好使最大初动能光电子不能到达阳极时所需的反向电压大小。

在实验记录和拟合中，我们记录的是截止电压的正值大小；在仪器调节中，外加电压可以显示为负值，用来表示反向电压。答辩时我们会强调：反向电压是一类外加电压状态，只有达到截止条件时，其绝对值才对应截止电压。

### 4. 五个滤光片对应波长的频率如何计算？有没有考虑滤光片带宽？

理论上频率由：

```text
ν = c / λ
```

计算得到。平台内置了 365 nm、405 nm、436 nm、546 nm、577 nm 五个滤光片档位及对应频率，用于教学仿真。

当前版本将滤光片近似为理想单色光，还没有加入滤光片带宽、汞灯谱线半宽和透过率曲线。这是一个简化边界。后续会把滤光片带宽作为误差模型的一部分加入。

### 5. 如果把入射光强调大，平台中截止电压会不会变化？为什么？

不会变化。理想光电效应中，截止电压只由入射光频率和阴极材料逸出功决定。光强变大表示单位时间入射光子数变多，所以逸出电子数量和饱和光电流会增大，但单个光子的能量 `hν` 不变，单个光电子最大初动能不变，因此截止电压不变。

平台也是这样实现的：光强影响光电流和动画事件密度，但不参与截止电压计算。

### 6. 如果光强变大，光电流如何变化？饱和光电流与光强在模型里是什么关系？

光强变大，单位时间照射到阴极的光子数增加，能够逸出的光电子数增加，因此光电流增大。在正向电压足够大、光电子基本都被阳极收集时，饱和光电流近似与光强成正比：

```text
Isat ∝ 光强
```

平台中饱和电流由光强比例决定，并受到光阑、距离等参数影响；当前是教学化模型，没有进一步加入空间电荷效应或真实阳极收集效率标定。

### 7. 如果入射光频率低于截止频率，微观动画和电流读数分别会发生什么？

如果频率低于截止频率，单个光子能量不足以克服材料逸出功，理论上不能产生光电子。平台中会出现：

- 理论截止电压为 0；
- 光电效应条件不成立，生成光电流为 0 或仅显示本底/偏置读数；
- 微观动画显示光子入射但能量低于阈值，电子不能真正逸出并形成被收集的光电流。

### 8. 铯、钾、钠、锌、铜等材料的逸出功取值来自哪里？是否与真实光电管材料一致？

平台使用的是常见教材或物理资料中的典型逸出功数量级，用于展示不同材料对截止频率和截止电压的影响。当前配置为教学仿真参数，并未完成与某一具体真实光电管材料的严格标定。

因此答辩时不能说这些参数完全等同于真实仪器。更准确的说法是：它们用于物理规律教学展示；后续如果要对接真实仪器，会依据实际光电管阴极材料和实验数据重新标定。

### 9. 对同一金属材料，改变光强和改变频率分别影响哪些物理量？

同一材料下：

- 改变频率：改变单个光子的能量 `hν`，从而改变光电子最大初动能、截止电压；若低于截止频率，则不能发生光电效应。
- 改变光强：改变单位时间入射光子数，从而改变逸出电子数量和饱和光电流；理想情况下不改变单个光电子最大初动能，也不改变截止电压。

这也是平台设计中最想让学生区分的核心关系。

### 10. 计算普朗克常量 h 是由哪条拟合曲线的斜率得到？斜率和 h 的关系是什么？

由截止电压 `Uc` 与入射光频率 `ν` 的线性拟合曲线得到：

```text
Uc = (h/e)ν - W0/e
```

拟合斜率：

```text
k = h/e
```

所以：

```text
h = k e
```

平台中横轴使用 Hz，纵轴使用 V，拟合斜率乘以元电荷 `e = 1.602176634e-19 C` 得到 `h`，再与理论值 `6.62607015e-34 J·s` 比较计算相对误差。

## 二、伏安特性曲线追问

### 11. 你们的 I-U 曲线是如何生成的？理论推导还是真实仪器数据经验拟合？

当前运行版采用教学化简化模型，不是严格真实仪器数据拟合。平台使用平滑响应函数模拟光电流随外加电压变化，从反向遏止区逐渐过渡到正向饱和区，并叠加少量噪声。

我们不会把它表述为已经完成真实仪器标定。后续计划用真实光电效应实验仪采集 I-U 数据，对曲线形状、拖尾、饱和区和零点漂移进行标定。

### 12. “基于费米-狄拉克分布的光电效应伏安特性拟合曲线，加入本地修正”具体是什么意思？

这个表述需要谨慎。当前运行版还没有把费米-狄拉克分布作为严格数值计算主模型。我们最初方案中讨论过用它描述金属中电子能级占据和电子初动能分布，但当前提交版本主要采用简化平滑模型表现伏安曲线的连续变化。

答辩时建议这样说：目前费米-狄拉克分布还属于后续物理真实性增强方向，当前版本用本地简化模型服务教学演示；我们会避免宣称已经完成严格费米-狄拉克拟合。

### 13. 真实光电管伏安曲线接近截止电压时不是突变到零，你们如何模拟拖尾？

平台用连续平滑函数模拟电流随反向电压逐渐减小的过程，而不是让电流在某一点突变为零。这样能够体现接近截止电压时仍可能存在少量电子到达、仪器本底和读数阈值等现象。

但当前拖尾是教学化平滑模型，不是来自真实仪器拖尾数据的严格拟合。

### 14. 动画中电子数量变化与实际电流数值有没有定量对应？

当前是趋势联动和半定量映射，不是严格逐电子计数。动画会读取当前光强、频率、电压和光电流状态，使电子发射密度、收集比例和宏观电流趋势一致；但不能把画面中一个电子直接等同于真实电子，也没有严格按 `I = Ne/t` 做逐粒子换算。

后续可以加入按电荷量换算的统计采样，使单位时间到达阳极的动画粒子数与电流建立更明确比例关系。

### 15. 你们如何定义“截止”？电流等于 0 还是低于阈值？

理论截止电压由光电方程直接计算。实验操作中，学生通过调节反向电压，使电流接近零，再记录该反向电压大小为测得截止电压。

平台内有接近零电流的容差判断，用于调零和记录流程。当前不是完整真实仪器判据，而是教学仿真中的“近零电流”判据。

### 16. 是否考虑空间电荷效应、阳极收集效率等因素？

当前运行版没有严格考虑空间电荷效应和阳极收集效率的真实物理模型。正向电压下电流趋于饱和主要由简化饱和电流模型控制。

答辩时应说明：这些因素是后续提升真实性的方向，目前平台重点服务基础规律教学。

### 17. 对不同频率的光，动画中电子速度是否随频率改变？

有体现。微观动画会根据光子能量与逸出功之差计算最大动能趋势，频率越高，电子初速度表现越快；频率低于阈频时，电子不能逸出形成有效收集。

但这仍是教学可视化映射，不是严格真实电子速度分布模拟。

### 18. 正向电压增大时，电流最终趋于饱和。平台饱和值由什么决定？

饱和值主要由光强决定，并受到光阑、距离和灯状态影响。平台中光强越大，饱和光电流越大；正向电压增大到一定程度后，电流趋于该饱和值。

当前没有把空间电荷、阳极几何收集效率等真实仪器因素加入饱和值计算。

## 三、阳极电流、负电流、本底修正

### 19. “阳极电流是造成测量截止电压不准确的主要原因”中的阳极电流具体指什么？

在真实光电管中，阳极电流可能指阳极受到杂散光照射产生的反向光电流，也可能与暗电流、漏电流、零点漂移等本底有关。它会让截止点附近的读数不再理想为零，从而影响截止电压判断。

当前平台没有严格区分阳极反向光电流、暗电流和漏电流的完整来源模型，只实现了仪器偏置电流、零点校验和测量噪声。因此答辩时应承认：目前是合并为本底/偏置误差的教学化处理，后续会细分来源。

### 20. “负电流”方向如何定义？为什么会是负值？

光电流方向取决于仪器约定。若正常光电子由阴极到阳极形成的电流被定义为正方向，那么由本底、阳极受光或零点漂移引起的反向读数可能显示为负值。

平台中初始仪器偏置电流可以为负，用来提示学生真实实验中零点不一定天然为 0，需要调零。当前不是完整阳极反向发射模型，而是偏置误差模拟。

### 21. 本底电流包括哪些来源？暗电流、环境光、电路零漂如何处理？

真实实验中本底可能包括暗电流、环境光、漏电流、电路零漂、阳极反向光电流等。当前平台主要实现了：

- 初始仪器偏置电流；
- 手动调零和零点确认；
- 记录时测量噪声；
- 零点基准偏差校验。

环境光、暗电流和阳极反向光电流没有被拆成独立物理模型，后续会作为误差模型细化。

### 22. 调零后仍有本底漂移，平台是让学生看到误差还是自动消掉？

平台不会简单把误差全部自动消掉。它保留了调零流程、零点基准和记录前校验；如果当前电流与零点基准偏差过大，会提示或阻止记录。

但当前漂移模型仍较简化，还没有实现随时间连续变化的真实零漂曲线。后续可以加入慢变漂移，让学生更直观看到真实测量的不确定性。

### 23. 一键调零和手动调零在物理意义上是否等价？

不完全等价。手动调零更接近真实实验操作，能够让学生理解零点误差和调零顺序。一键调零如果使用，只适合作为教学演示或快速复位功能，不应替代学生理解真实调零过程。

当前平台强调记录前的调零确认和零点校验，避免学生完全忽略调零误差。

### 24. 如果故意不调零或量程选错，会出现怎样的数据偏差？AI 如何判断？

不调零时，平台会提示先调零、接线和确认零点；若零点偏差过大，可能阻止记录或造成截止电压判断偏差。量程选错时，电流显示可能不易读或出现溢出，导致学生判断截止点不准确。

AI 助手不是凭空判断，而是结合操作状态、原始记录、拟合结果和规则诊断：例如截止电压是否随频率增大而增大、普朗克常量误差是否过大、重复频率是否过多、数据量是否不足等。

## 四、微观可视化细节

### 25. 动画每一步是否都有对应物理判据？

主要步骤有对应的教学判据：

- 光源和盖子状态决定是否有入射光；
- 频率与逸出功比较决定是否能逸出；
- 光强决定事件密度；
- 频率决定最大动能趋势；
- 外加电压决定加速、减速、部分收集或截止表现。

但动画是教学化表达，不是逐粒子真实量子动力学模拟。

### 26. 如何避免“光子像弹珠一样撞出电子”的误解？

答辩时要主动说明：动画不是经典小球碰撞，而是把不可见的量子吸收过程模型化呈现。真实光电效应中电子吸收光子能量，不是宏观弹性碰撞。动画的作用是帮助学生理解能量守恒和阈频条件，而不是还原真实微观轨迹。

后续也可以在界面说明文字中进一步强化“模型化示意”的边界。

### 27. 频率不够时电子是否完全不逸出？

在当前教学模型中，如果频率低于材料截止频率，电子不会形成有效逸出和收集，光电流为 0 或仅保留本底读数。动画会显示阈频判断失败，不产生有效光电子流。

### 28. 频率刚刚高于截止频率，是否体现电子速度较慢、容易被反向电压阻止？

有体现。频率刚高于阈频时，最大动能较小，动画中的电子速度较慢，反向电压更容易使其折返或无法到达阳极。

但这种速度表现是教学映射，不是严格按真实初动能分布逐粒子计算。

### 29. 反向电压改变时，动画轨迹如何变化？是否基于 `F = eE` 计算？

动画会根据反向电压大小改变电子收集比例、减速表现和轨迹状态，如“部分收集”“飞行后折返”“截止”等。

当前没有做完整电场分布和 `F = eE` 的精确数值积分，而是把电压映射为动画中的加速度和收集门控。答辩时应明确这是教学化近似。

### 30. 是否体现随机性？

有一定随机性。动画中的光子、电子事件位置、散射和收集有随机采样；记录数据时也加入了测量噪声。

但还没有完整模拟真实电子初动能方向和大小的统计分布。

### 31. 光电子初动能分布是否模拟？还是都按最大初动能处理？

当前主要按最大初动能和简化概率映射来控制动画表现，没有严格模拟完整初动能分布。部分电子被阻止、部分电子到达的连续变化是通过收集概率和电流门控体现的。

这是后续物理模型增强重点之一。

### 32. 截止电压附近是否体现“部分电子被阻止、部分电子到达”？

在接近截止区域，动画能体现部分收集、折返和逐渐截止的趋势，伏安曲线也不是突变。但它不是严格的能量分布积分模型，当前属于定性到半定量的教学可视化。

### 33. 微观动画与宏观电流读数有没有数学映射？是否体现 `I = Ne/t`？

目前没有严格逐电子计数到电流的换算。动画读取宏观状态，使粒子密度和收集比例与电流趋势一致，但没有把每个动画粒子按电子电荷量换算为真实电流。

可答：当前是趋势一致的半定量映射，后续计划加入基于 `I = Ne/t` 的统计采样。

## 五、数据拟合和普朗克常量测量

### 34. 至少需要几组不同频率数据？数据点不足如何提示？

平台至少需要两种不同频率的截止电压数据才能进行线性拟合并估算普朗克常量。界面会提示“至少记录两种不同频率的测得截止电压后，才能计算普朗克常量”。

从实验教学角度，我们建议记录 3 到 5 组不同频率数据，拟合结果更稳定。

### 35. 斜率、截距分别对应什么物理意义？

拟合关系：

```text
Uc = (h/e)ν - W0/e
```

- 斜率 `k = h/e`，所以 `h = ke`；
- 纵轴截距 `b = -W0/e`，与材料逸出功有关；
- 横轴截距对应截止频率 `ν0`。

### 36. 从截距如何得到逸出功？从横轴截距如何得到截止频率？是否一致？

由：

```text
b = -W0/e
```

得到逸出功。用 eV 和 V 表示时，`|b|` 的数值可近似对应逸出功 eV。

截止频率：

```text
ν0 = W0 / h
```

理论上二者应与同一条拟合直线一致。实际数据中如果有误差，二者会有偏差，这正是误差分析的一部分。

### 37. 相对误差标准值取自哪里？

平台使用理论普朗克常量：

```text
h = 6.62607015e-34 J·s
```

作为标准值计算相对误差。当前不是某台仪器的标定值。

### 38. 如果学生记录了错误截止电压，平台直接拟合还是剔除异常点？

当前平台主要保留原始数据并直接用于拟合，不自动删除学生数据。学生可以在历史记录中删除明显错误点。AI 诊断会提示趋势异常、拟合质量差、误差过大等问题，但不应未经学生确认自动剔除。

这个设计是为了保留实验真实性和可复核性。

### 39. AI 诊断异常数据依据统计离群点还是物理规律？

两者都有，但以物理规律为主。当前诊断会检查：

- 截止电压是否随频率增加而增加；
- 截止电压是否出现负值或异常偏大；
- 拟合质量是否较差；
- 普朗克常量相对误差是否过大；
- 是否存在重复频率或数据量不足。

AI 的作用是把这些规则诊断转化为可读解释，不直接替代学生判断。

### 40. 某一波长数据偏离直线，如何判断原因？

我们会先从操作和数据角度判断：

1. 是否调零不准确；
2. 是否记录了错误截止点；
3. 是否滤光片档位选错；
4. 是否数据量不足或重复记录；
5. 是否模型误差或滤光片理想化假设导致偏差。

当前平台还没有真实滤光片误差和真实仪器标定数据，所以不能直接判定是滤光片本身误差。更稳妥的回答是：平台先提示可能原因，最终需要学生结合原始记录复核。

### 41. 是否给出拟合不确定度、标准误差、R²？

当前报告主界面主要给出 `h`、相对误差、逸出功和截止频率。AI 诊断模块中会计算拟合质量相关指标并用于提示，但斜率标准误差、截距标准误差和完整不确定度传播还没有在报告中系统展示。

后续会把 R²、斜率标准误差、截距标准误差和不确定度传播加入实验报告。

### 42. 是否考虑接触电势差？

当前没有加入接触电势差模型。原因是本版本主要面向基础教学，先保证学生理解 `Uc-ν` 线性关系、斜率求 `h` 和截距求逸出功。

接触电势差会影响截止电压截距，是后续提高真实性时需要加入的重要修正项。

## 六、高精度虚拟仪器质疑

### 43. 电压调节精度达到 0.001 V，是否等价于实验精度提高？

不等价。虚拟仪器的调节分辨率提高，只说明参数可调得更细；实验精度还取决于误差模型、零点漂移、读数噪声、滤光片误差、光源稳定性等。

因此答辩时应说：平台提供高分辨率调节能力，但不能把它直接等同于真实实验精度提高。

### 44. 如果数据由公式生成，高精度是否只是程序设定？

是的，需要承认这一点。当前平台的数据主要来自理论模型和教学化误差模型，不是通过真实仪器标定得到的高精度测量能力。

我们的价值不在于声称比真实仪器更精密，而在于让学生可控地观察物理规律、操作流程和误差影响。

### 45. 是否人为加入真实仪器误差？

已加入一部分简化误差：

- 仪器初始偏置电流；
- 电流调零和零点确认；
- 测量电流随机扰动；
- 光电流噪声；
- 量程溢出和显示限制。

尚未加入或未完整实现：

- 滤光片带宽；
- 汞灯稳定性；
- 环境光；
- 暗电流独立模型；
- 接触电势差；
- 空间电荷效应；
- 真实仪器标定曲线。

### 46. 如果没有随机误差，学生会不会误以为真实实验总能得到完美直线？

平台目前已经加入了一些噪声和操作误差提示，但还不够完整。我们在答辩中会强调：真实实验不可能总是完美直线，平台后续会继续加强随机误差和系统误差模型。

### 47. 如何平衡教学友好和实验真实性？

我们的策略是分层：

- 基础层：保证学生能清楚看到核心物理规律；
- 操作层：保留开灯、开盖、调零、接线、换滤光片、调电压、记录数据等真实流程；
- 误差层：逐步加入本底、噪声、量程和拟合偏差；
- 提升层：后续加入真实仪器标定、接触电势差、滤光片带宽等。

所以当前平台更偏教学友好，同时保留部分实验真实性。

### 48. 是否用真实 ZKY-GD4 仪器数据验证过？误差是多少？

从当前运行项目看，还没有完成与 ZKY-GD4 真实仪器数据的系统标定，因此不能给出真实验证误差。

答辩时应如实回答：目前主要完成理论模型和教学误差模型验证，后续会采集真实仪器 I-U 曲线和截止电压数据，用于修正模型参数。

## 七、教学价值和创新性

### 49. 核心物理创新点是什么？

核心不是单个公式或单个动画，而是把实验操作闭环、数据拟合、2D/3D 同步、微观机制可视化和 AI 辅助诊断结合起来。学生不仅看到公式结果，还能经历完整实验流程，并观察参数变化对宏观数据和微观示意的同步影响。

### 50. 知识图谱中概念如何关联？举一个学习路径例子。

知识图谱把光电效应、截止频率、截止电压、饱和光电流、逸出功、普朗克常量等概念连接起来。

例子：学生点击“截止电压”，可以看到它与光电子最大初动能有关，再进一步理解 `eUc = Ekmax`，再连接到 `hν = W0 + Ekmax`，最后理解为什么测量 `Uc-ν` 直线斜率可以求普朗克常量。

当前知识图谱是项目内手工构建的数据，不是运行时由 AI 自动生成。

### 51. 如果学生已经理解公式，平台还能带来什么额外价值？

平台提供公式之外的实验操作体验和误差体验。学生可以亲自调节滤光片、光强、电压、材料、距离和光阑，记录数据并观察拟合结果。微观动画帮助他们区分“频率影响能量”和“光强影响数量”。AI 和报告模块则帮助学生把数据、图像和误差分析串联起来。

### 52. 微观动画会不会过度经典化？有没有提示？

有这个风险，所以答辩时必须主动说明动画是模型化教学表达，不是完整量子过程模拟。当前动画和文字已经强调频率、光强、电压的物理关系，但后续还会进一步增加“示意模型”提示，避免学生误解为真实电子轨迹。

### 53. 是否支持教师自定义参数？

当前运行版没有提供完整教师端自定义参数面板。材料、滤光片、光阑、量程等参数是项目中预设的，学生可以在界面中选择，但教师不能在页面上直接自定义逸出功、误差大小或盲测数据。

这是后续可扩展方向。

### 54. 如何评价教学效果？有没有学生测试、对照实验或问卷？

当前项目代码和文档中没有看到系统学生测试、对照实验或问卷数据。因此不能声称已经完成教学效果量化验证。

## 八、计算机、AI、建模问题

### 56. 前后端分离中，关键物理计算在前端还是后端？

以当前运行版为准，关键物理计算主要在前端完成，集中在 Vue + Pinia 的实验状态模型和辅助函数中。AI 问答通过外部 DeepSeek API 调用，但它不负责核心物理计算。

如果材料中写“后端负责物理逻辑计算”，答辩时要修正为：当前版本没有独立后端物理服务，核心计算在前端确定性模型中完成。

### 57. 网络断开后，平台还能仿真实验和拟合吗？AI 是否依赖联网？

仿真实验、数据记录、I-U 曲线、`Uc-ν` 拟合和报告基础生成都可以在本地前端运行。AI 在线问答依赖 DeepSeek API，需要联网；API 不可用时项目里有本地兜底回答，但能力较简单。

### 58. AI 助手如何保证误差分析不胡编？

不能说完全保证。我们的做法是让核心物理计算由确定性程序完成，AI 只读取原始数据、拟合摘要和规则诊断结果，再生成解释和建议。平台保留原始数据、理论截止电压和偏差，便于学生和教师复核。

答辩时应说：AI 是辅助，不是最终物理判据。

### 59. 知识图谱是手工构建还是 AI 自动生成？

当前项目中的知识图谱数据是手工构建在本地数据文件中的，不是运行时由 AI 自动生成。因此概念关系相对可控。后续如果引入 AI 生成知识关系，需要教师审核后才能进入正式知识库。

### 60. 3D 模型中的旋钮、按钮与物理参数如何映射？

3D 页面通过 Spline 场景对象名称与交互事件映射到 Pinia 状态。比如滤光片旋钮对应不同波长状态，调节后写回 `filterWavelength`；量程旋钮写回 `currentRange`；调零旋钮写回 `zeroAdjust`；电压按钮会改变外加电压；盖子和接线状态也同步到同一状态源。

这样 2D 页面、3D 场景、报告区和微观动画读到的是同一套实验状态。

### 61. 2D 和 3D 同时操作会不会出现状态冲突？

当前是单页面应用中的统一 Pinia 状态源，一般不会出现两个独立后端同时写入的并发冲突。状态更新带有来源标记和 revision，用于同步显示。

如果未来支持多窗口、多用户协同或服务端同步，就需要加入更严格的并发控制。当前版本还没有做到多人协同层面。

### 62. AI 自动生成实验报告是否会把错误数据包装成正确结论？是否保留原始数据？

平台会保留原始记录，并在报告中展示原始数据、拟合数据、理论截止电压和偏差。AI 可以辅助解释，但不会删除原始数据，也不应该把错误数据包装成正确结论。

如果数据异常，AI 应提示可能原因，例如调零不准、滤光片选择错误、数据量不足或拟合误差过大。

## 九、刁钻现场追问

### 63. “量子性在实验中无法被看到”，动画中看到的是量子性还是经典化示意？

动画中看到的是对量子机制的经典化、模型化示意，不是真实量子过程本身。它服务于教学理解，帮助学生把能量量子化、阈频和截止电压联系起来。

### 64. 光电效应证明光具有粒子性，但不是说光只具有粒子性。平台有没有体现波粒二象性边界？

当前平台重点展示光电效应对光量子性的支持，即单个光子能量 `E = hν`。它没有系统展示波动性实验，例如干涉、衍射等。

答辩时可说：平台聚焦光电效应，不把它扩展成完整波粒二象性平台；后续可通过知识图谱补充边界说明。

### 65. 频率不变、光强极低时，是否考虑单光子、单电子概率发射？

当前没有严格模拟单光子、单电子层面的概率发射。平台的光强主要控制宏观事件密度和饱和电流，动画中有随机性，但不是单光子探测级别的量子统计模型。

这是后续高级模式可以扩展的方向。

### 66. 是否区分“光电子逸出数量”和“单个光电子最大初动能”？

这是平台重点区分的内容：

- 光强影响光电子逸出数量和饱和光电流；
- 频率影响单个光电子最大初动能和截止电压。

界面、动画和答辩都要避免说“光强越大电子能量越大”。

### 67. 为什么测普朗克常量用截止电压与频率关系，而不是光电流与光强关系？

因为普朗克常量出现在单个光子能量公式 `E = hν` 中。截止电压反映单个光电子最大初动能：

```text
eUc = hν - W0
```

所以 `Uc-ν` 的斜率直接给出 `h/e`。而光电流与光强关系主要反映单位时间光电子数量，不直接给出单个光子的能量，因此不能用来直接测 `h`。

### 68. 如何证明平台不是“更漂亮的公式计算器”？

平台不仅计算公式，还包含完整实验流程：开灯、开盖、调零、接线、换滤光片、调电压、记录数据、自动扫描、历史记录、曲线拟合、报告生成、AI 诊断、2D/3D 同步和微观动画。学生需要通过操作得到数据，再用数据拟合物理量。

因此它的价值在于实验闭环和可视化理解，而不只是公式输出。

### 69. 如果把铜作为阴极，再选 577 nm 入射光，会不会产生光电流？为什么？

不会产生有效光电流。577 nm 对应频率约 `5.196e14 Hz`，光子能量约 `2.15 eV`；铜的逸出功约 `4.65 eV`，光子能量低于逸出功，不能使电子逸出。

平台中这种情况下截止电压为 0，光电效应条件不成立。

### 70. 材料中出现 Unity、Blender、Spline，最终三维仿真主链路是什么？是否影响表述严谨性？

最终运行主链路是 Spline 场景文件 + 本地 Spline runtime + Vue 组件 + Pinia 状态源。Blender 或 Unity 可以作为建模、方案探索或素材处理阶段的工具，但答辩时应以最终运行链路为准。

为保证表述严谨，建议答辩时说：“当前网页端 3D 仿真主链路是 Spline + Vue + Pinia。”

## 十、连续追问速答版

### 1. 如何由爱因斯坦光电方程得到截止电压？

由 `hν = W0 + Ekmax` 和 `eUc = Ekmax` 得到：

```text
Uc = (h/e)ν - W0/e
```

平台按这个公式计算理论截止电压，低于阈频时取 0。

### 2. 光强改变时，截止电压为什么不变？

光强改变的是单位时间光子数，不改变单个光子的能量 `hν`。单个光电子最大初动能由频率和逸出功决定，所以截止电压不随光强变化。

### 3. 伏安曲线不是简单直线，具体如何拟合？

当前运行版使用教学化平滑响应模型模拟 I-U 曲线，不是严格真实仪器数据拟合。它能表现反向遏止、拖尾和正向饱和趋势。后续会采集真实仪器曲线进行标定。

### 4. 接近截止电压时为什么电流不是突然变为零？

真实电子初动能和出射方向有分布，仪器还有本底、噪声和读数阈值，所以电流接近截止点时是连续趋近，而不是理想突变。平台用平滑模型模拟这一现象。

### 5. 动画中电子到达数量与电流读数有没有定量关系？

当前是趋势一致的半定量映射，不是严格逐电子 `I = Ne/t` 计数。后续会加入更明确的统计换算。

### 6. 高精度数据来自真实仪器标定还是理想公式生成？

当前主要来自理论公式和教学化误差模型，不是完整真实仪器标定。我们不把它宣传成替代真实精密测量，而是用于教学仿真和规律理解。

### 7. 学生操作错误时，AI 如何区分误差和真实物理现象？

AI 基于原始数据、拟合结果和规则诊断提示问题，如调零不准、频率趋势异常、数据量不足、重复频率、普朗克常量误差过大等。最终判断仍由学生和教师复核。

## 十一、答辩中应避免的表述

### 不要说：“我们真实看到了量子过程。”

建议说：“我们将不可见的微观机制进行了模型化可视呈现。”

### 不要说：“虚拟仪器比真实仪器更精确。”

建议说：“虚拟平台提供更高分辨率的参数调节，并通过可控误差模型帮助学生理解测量过程。”

### 不要说：“AI 可以自动完成误差分析并给出最终结论。”

建议说：“AI 基于原始数据、拟合结果和预设规则给出误差来源建议，最终结论仍需学生判断和教师复核。”

### 不要说：“动画完全符合真实电子运动。”

建议说：“动画遵循主要物理规律进行教学化表达，不等同于完整量子动力学模拟。”

### 不要说：“光强越大，电子能量越大。”

正确说法：“光强越大，单位时间入射光子数越多，逸出电子数和饱和光电流增大；单个光电子最大初动能由频率决定。”

## 十二、最后总结

我们平台当前最稳妥的答辩定位是：基于爱因斯坦光电方程的教学仿真平台，结合简化伏安曲线、调零误差、数据拟合、2D/3D 同步和微观动画，帮助学生理解光电效应核心规律。

需要明确承认的边界是：当前没有完成真实仪器系统标定，没有完整实现滤光片带宽、接触电势差、空间电荷效应、严格费米-狄拉克分布和逐电子量子统计模型。这些不是答辩中的缺陷回避点，而是后续物理真实性增强方向。
