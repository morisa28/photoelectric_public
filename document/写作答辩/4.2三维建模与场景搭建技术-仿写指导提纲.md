本文用于指导后续撰写“光电效应实验模拟仿真平台”项目报告中的 `4.2 三维建模与场景搭建技术` 板块。写作时应以 PDF 原报告中该板块为模板，但必须结合 `event/new(20)` 项目的真实开发情况改写，不能照搬密立根油滴、Blender 导出 FBX、Unity 导入等与当前项目不一致的内容。

## 一、整体仿写原则

### 1. 保留 PDF 的章节骨架

PDF 原板块的组织方式是：

```text
4.2 三维建模与场景搭建技术

  1. 场景规划与布局

  1. 模型检查与优化
  2. 模型导出

```

仿写时建议保留这种“总标题 + 三个二级小节 + 条目化说明 + 图题”的结构。当前项目可以扩展为：

```text
4.2 三维建模与场景搭建技术
```

如果正文篇幅有限，也可以只写到 `4.2.3`，把 `4.2.4` 和 `4.2.5` 合并进“技术要点与经验总结”。如果报告需要突出项目工程量，建议保留 `4.2.4`，因为当前项目最有价值的内容不只是模型展示，而是 Spline 场景、Vue 状态和 3D 交互的集成。

### 2. 保留 PDF 的表达口吻

PDF 原文不是开发日志，而是项目报告。仿写时应使用“技术说明 + 目的说明 + 结果说明”的正式口吻。

推荐句式：

- “本项目参考真实光电效应实验仪器，将三维场景划分为……”
- “通过合理命名关键对象，便于后续在网页运行时进行定位、交互和状态同步。”
- “该设计既保证了三维场景的沉浸感，也降低了前端维护复杂度。”
- “显示屏采用运行时 Canvas 纹理刷新方式，使电流、电压读数能够与实验状态实时一致。”

不建议写成：

- “我把 SplineScene.vue 里某个函数改了……”
- “这一轮修了 bug……”
- “之前哪里坏了，现在修好了……”

项目报告中可以提“经过多轮迭代后，项目将 3D 主链路收口为 Spline 场景资源”，但不要写成流水账。

### 3. 必须替换 PDF 中不适用的对象

PDF 原板块写的是密立根油滴实验，当前项目是光电效应实验。仿写时要做如下替换：

| PDF 原内容 | 当前项目应改为 |
| --- | --- |
| 密立根油滴实验仪器 | 光电效应实验仪器 |
| 油滴腔体、电极、观察筒 | 光源、滤光片、光电管、控制面板、显示屏 |
| Blender 基础建模 + Unity 导入 | Spline 场景制作 + WebGL runtime 加载 |
| `.fbx` / `.obj` 导出 | `resource/scene.splinecode` 集成 |
| Unity 主相机、Directional Light | Spline 场景相机、网页运行时视角、诊断参数 |
| 油滴、电场力、刚体 | 光电流、电压、滤光片、光电管距离、状态同步 |

可以保留“模型检查、命名、分组、材质、运行效率”等通用表述，但要把具体技术链路改成当前项目真实链路。

## 二、各小节应如何仿写

### 4.2 三维建模与场景搭建技术

这一段是总述，建议写 2-3 段。

应写内容：

- 项目名称：光电效应实验模拟仿真平台。
- 三维场景目的：让用户在网页中直观看到光源、滤光片、光电管、控制面板和实验读数。
- 技术路线：Vue 3 + Pinia + Spline runtime + `scene.splinecode` + Canvas 纹理。
- 与 PDF 的对应关系：PDF 强调 Blender/Unity 建模，本项目应强调 Spline/WebGL 网页运行和状态同步。

建议补充文字：

```text
本项目的三维建模与场景搭建围绕光电效应实验教学目标展开，通过 Spline 构建实验台、光源、滤光片、光电管、控制面板和双显示屏等核心对象，并将场景导出为 `scene.splinecode` 资源集成到 Vue 前端页面中。网页端通过 `@splinetool/runtime` 加载三维场景，再结合 Pinia 实验状态实现电压、电流、滤光片和光电管距离等参数的实时同步。
```

```text
参考真实光电效应实验装置的操作顺序，场景将光源、滤光片和光电管沿光路方向布置，使用户能够直观看到入射光从光源发出、经过滤光片筛选并进入光电管的过程。控制面板集中放置在实验台正面，便于用户完成电源切换、电压调节、测量模式切换、量程选择和调零确认等操作。
```

```text
三维实验页面通过 `/3d-view` 路由独立承载，页面层由 `ThreeDView.vue` 提供工具栏、数据弹窗和教学小窗入口，场景层由 `SplineScene.vue` 加载 `resource/scene.splinecode`。为便于开发和验收，页面还支持诊断模式、等待时长、缩放比例和背景色等 URL 参数，使场景截图和对象检查过程更加稳定。
```

#### 3. 场景细节与优化

应写内容：

建议补充文字：

```text
场景关键对象均保留明确名称，便于网页运行时查找和状态绑定。例如 `left_plane` 与 `right_plane` 分别作为电流、电压显示屏，`Phototube_Box` 用于光电管拖拽，`Filter_Rotator` 用于滤光片旋转，`Knob_CurrentRange` 与 `Knob_ZeroAdjust` 用于量程和调零旋钮状态同步。
```

### 4.2.3 模型导出与集成准备

这一节是最容易误写的地方。不能继续写“导出 `.fbx` 或 `.obj` 供 Unity 使用”，而应改为“导出 Spline 场景资源并通过 Vite 集成”。

#### 1. 模型资源检查与优化

应写内容：

- 检查对象命名。
- 检查显示屏材质是否支持运行时贴图。
- 检查交互对象是否可命中。
- 检查场景体积是否适合网页加载。

建议补充文字：

```text
模型集成前需要检查关键对象是否具备稳定名称和可访问层级，尤其是显示屏、按钮、旋钮、滤光片和光电管等交互对象。同时需要确认显示屏材质能够在运行时插入纹理层，交互区域具备足够明确的命中范围，场景体积也应控制在浏览器可接受的加载范围内。
```

#### 2. 场景导出与网页集成

应写内容：

- Spline 导出 `scene.splinecode`。
- 放入 `resource/scene.splinecode`。
- Vite 通过 `?url` 引入。
- `SplineScene.vue` 使用 `Application.load(sceneUrl)` 加载。

建议补充文字：

```text
三维场景在 Spline 中完成后导出为 `scene.splinecode` 文件，并放置于项目 `resource` 目录。前端通过 `import sceneUrl from '../../resource/scene.splinecode?url'` 获取构建后的资源地址，再由 Spline runtime 的 `Application` 实例加载场景，实现从建模工具到网页端 WebGL canvas 的集成。
```

#### 3. 技术要点与经验总结

应写内容：

- 早期项目曾有 Three.js/GLB/HDR 多资源链路。
- 当前主链路已收口为 Spline。
- 网页端重点维护状态同步、材质刷新、交互映射、诊断脚本。

建议补充文字：

```text
项目早期曾采用 Three.js、本地 GLB 模型和 HDR 环境贴图组合场景，灵活度较高，但模型装配、交互命中、材质更新和状态同步都集中在前端代码中，维护成本较高。后续迭代将 3D 主链路收口为 Spline 场景资源，使模型细节主要在 Spline 中维护，网页端专注于状态同步、材质贴图刷新、交互事件映射和自动化诊断。
```

### 4.2.4 场景运行与交互集成

这一节是建议新增内容，用来体现当前项目比 PDF 模板更完整的技术流程。

#### 1. 统一实验状态驱动

应写内容：

- Pinia `experiment` store 是唯一事实来源。
- 2D 和 3D 共用参数。
- 3D 操作只触发 store action。

建议补充文字：

```text
本项目的 2D 页面与 3D 页面共用同一个 `experiment` store。电压、频率、光强、滤光片波长、光电管距离、仪器电源、工作模式、测量模式、量程和数据记录等内容均由 store 统一管理。3D 场景中的按钮、旋钮和拖拽操作只负责触发 store action，显示层再根据状态变化刷新模型表现。
```

#### 2. 控制面板交互实现

应写内容：

- 按钮点击。
- 旋钮拖拽。
- 滤光片旋转。
- 光电管拖拽。
- 线缆连接状态。

建议补充文字：

```text
3D 控制面板通过射线检测和热区兜底机制识别用户操作。按钮点击用于触发电压增减、模式切换、电源开关和调零确认；旋钮拖拽用于调整量程和调零值；滤光片旋转用于切换滤光片波长；光电管拖拽用于改变光电管距离。所有操作结果最终写回 Pinia store，并同步刷新 2D 页面、3D 屏幕和数据记录状态。
```

#### 3. 数据记录与教学小窗集成

应写内容：

- 3D 页面可记录当前数据。
- 可查看数据曲线。
- 微观原理小窗是页面浮层，不是 Spline 内部模型。
- Canvas 2D 用于解释光子、电子、电场收集过程。

建议补充文字：

```text
3D 页面顶部工具栏提供数据记录和数据查看功能，使用户可以在三维场景中完成实验操作后直接保存当前数据并查看曲线分析结果。页面还集成了微观原理演示小窗，该小窗与 Spline 场景同级挂载，通过 Vue 组件和 Canvas 2D 动画展示光子入射、能量吸收、阈频判断、电子逸出和电场收集等过程。
```
