# 当前代码适配版电流逻辑实施方案-new(26)

## 0. 文档定位

本文档用于整合以下两份电流逻辑修复指导中的有效意见，并按当前项目实际状态重新制定实施方案：

- `photoelectric_current_logic_fix_guide.md`
- `current_version_current_logic_fix_guide.md`

这两份文档中关于“旧版本存在 `tanh` 主电流模型、阴极主光电流会变负、3D 微观小窗与 store 口径不统一”的问题，在当前本地版本中已经不再完全成立。因此本文不照搬旧方案，而是根据当前代码和用户确认的 14 项方向，形成第三版“当前代码适配版”方案。

本文当前阶段只作为实施方案，不直接修改源码。

## 1. 用户已确认的方向

### 1.1 修改范围

本轮后续如进入实现，范围限定为：

```text
只处理截止电压模式。
伏安特性模式尽量不动。
```

因此不应重构当前伏安特性曲线，不应替换现有非包围式模型，也不应为了截止模式去影响 U95/U99、正向收集率、自动伏安扫描等链路。

### 1.2 不进行回退

本轮不做“回退至物理框架大改之前”。

后续方案应基于当前物理框架继续收敛，而不是恢复旧公式、旧数据流或旧 UI。

### 1.3 截止电压记录口径

截止电压记录继续以用户当前手动调到的反向外加电压为核心：

```text
记录截止电压 = 当前外加反向电压的绝对值
```

不新增自动求解 `I_total = 0` 的表观截止电压流程。

### 1.4 拟合数据来源

普朗克常量拟合默认使用：

```text
用户手动记录且质量合格的截止电压
```

理论截止电压只作为对照，不直接替代实验记录；表观零电流点也不作为默认拟合值。

### 1.5 阳极寄生电流

当前 3D 默认较明显的阳极寄生保留。

这意味着后续不把默认阳极污染强度改成“接近无影响”的理想状态，也不把严重污染强制拆成新增 UI 模式。

但需要注意：保留阳极寄生不等于让它决定截止电压拟合。截止电压拟合应继续依赖用户手动记录和质量判定，而不是自动用总测量电流零点替代用户操作点。

### 1.6 3D 电流显示屏

3D 左侧电流显示屏继续显示仪器读数：

```text
仪器读数 = 物理测量电流 + 仪器零漂/调零基准
```

它不改成只显示真实阴极电流，也不改成只显示物理测量电流。

### 1.7 暗电流和本底电流

暗电流、本底电流只保留在原理动画演示中。

3D 实验仪器读数不新增暗电流和本底电流。

### 1.8 UI 与诊断显示

不新增 UI。

已有的数据显示、诊断、记录字段可保留；原来没有的侧栏、提示、分量面板，本轮不新增。

### 1.9 光强口径

实验仪器和 3D 区继续使用：

```text
光阑面积比例 × 距离平方反比
```

原理动画继续使用手动光强百分比。

二者保持区分。

### 1.10 正常操作下的目标

当用户按正常实验流程操作时，Cs 材料和五个滤光片下的截止记录应接近理论值。

如果正常操作下误差明显过大，说明实验设计、默认状态、记录判定或操作引导仍存在问题，需要综合排查。

这里不把某个百分比写成程序硬阈值；“接近理论值”应通过物理模型、实验流程、质量判定、默认状态和用户操作共同达成。

## 2. 对两份旧指导文档的综合判断

### 2.1 可吸收的部分

两份文档中以下判断仍然有价值：

1. 电流必须分层理解：

```text
真实阴极电流
阳极寄生电流
物理测量电流
仪器基准电流
最终仪器读数
```

2. 正常阴极光电流必须非负。

3. 阳极寄生电流应是独立分量，而不是混在阴极主光电流中。

4. 仪器零漂和调零基准只属于仪器读数口径，不属于真实光电流。

5. 截止电压记录不能简单等同于“用户看到电流表为零”的时刻；电流表零点可能受阳极寄生和调零基准影响。

6. 实验记录中应保留足够字段，方便区分理论截止电压、手动记录电压、电流分量和数据质量。

### 2.2 不应直接采用的部分

以下内容不适合直接用于当前项目：

1. 不再按“当前仍存在 `tanh` 旧模型”作为问题前提。

当前版本已经使用非包围式收集率模型，阴极主光电流不再由一条可正可负的旧曲线生成。

2. 不新建一套完全独立的电流模型替换现有物理内核。

当前已有统一物理内核。后续应在现有内核和 store 派生链路上做小范围修正，而不是另起一个会重复职责的模型文件。

3. 不采用旧指导中高零偏收集率模型。

旧文档建议的零偏压收集率较高，会明显改变当前非包围式光电管伏安曲线。用户已确认保留当前非包围式模型。

4. 不把光强简化为 `光强百分比 / 100` 来替代实验仪器的光阑和距离模型。

实验仪器使用光阑面积和距离平方反比；原理动画使用百分比。二者继续区分。

5. 不新增自动表观截止点求解。

用户已确认不需要根据 `I_total = 0` 自动求解表观截止电压。

6. 不新增 UI 诊断面板。

已有内容保留，原来没有的显示项不新增。

## 3. 当前框架应继续保留的部分

### 3.1 阴极主光电流

继续保持：

```text
I_cathode >= 0
```

反向电压达到理论截止条件时：

```text
U <= -Vs
I_cathode = 0
```

### 3.2 非包围式收集率

继续保留非包围式光电管模型：

```text
I_cathode = Is · C(U)
```

其中：

- `Is` 由相对光通量决定。
- `C(U)` 由外加电压、截止电压、正向上升尺度等决定。
- 光强改变电流规模，不改变理论截止电压。

### 3.3 阳极寄生电流

继续保留当前阳极寄生分量：

```text
I_measured = I_cathode + I_anode
```

其中 `I_anode` 在反向电压下可能为负。

### 3.4 3D 电流屏

继续保留：

```text
I_meter = I_measured + I_reference
```

未接入后方电流线时：

```text
I_meter = I_reference
```

这符合用户确认的 3D 电流屏口径。

## 4. 真正需要进一步处理的问题

综合两份指导文档和用户确认方向，当前真正需要进一步处理的不是“整体电流模型重构”，而是以下几个更窄的问题。

### 4.1 截止模式记录口径需要更明确

当前记录仍以当前外加反向电压的绝对值作为截止电压。这一口径可以保留，但文档、记录字段和质量判定应明确：

```text
这是用户手动记录的实验截止电压。
它不是自动求解的表观零电流点。
它也不是直接替代理论截止电压。
```

### 4.2 电流表零点不能等同于真实截止点

由于 3D 电流屏显示的是：

```text
物理测量电流 + 仪器基准
```

因此用户看到电流屏接近 0，不必然表示真实阴极光电流已经截止。

截止电压模式应继续依赖：

- 当前外加反向电压。
- 理论截止电压对照。
- 阴极收集率是否足够接近 0。
- 记录质量判定。

而不是直接用仪器显示电流求零点。

### 4.3 正常操作误差过大时要查实验设计

用户确认：正常操作下默认截止记录应接近理论值。

因此如果出现五个滤光片整体偏离理论值的情况，不应简单解释为“阳极寄生导致正常偏差”，而应检查：

1. 用户是否按正确流程调节到截止附近。
2. 记录质量判定是否过宽或提示不足。
3. 默认电压步进和显示精度是否足够支持低截止电压场景。
4. 默认阳极寄生是否在截止模式下过度影响用户判断。
5. 仪器零点和后方接线逻辑是否让用户误把仪表零点当成真实截止。
6. 记录和拟合是否只使用质量合格数据。

这里不在程序中写死某个百分比阈值，但误差显著偏大时应视为模型和流程需要继续修正。

## 5. 后续建议实施方案

### P0：不改源码，仅完成方案与数据核对

当前阶段已完成：

- 明确用户方向。
- 排除旧文档中过时前提。
- 形成当前代码适配版方案。

下一步若继续推进，应先用当前模型跑一组标准场景数据，确认偏差来源。

建议标准场景：

```text
材料：Cs
滤光片：365 / 405 / 436 / 546 / 577 nm
光阑：4 mm
距离：40 cm
测量模式：截止电压
阳极寄生：保持当前默认
暗电流/本底电流：不进入 3D 仪器
```

需要输出：

- 理论截止电压。
- 该理论截止电压附近的真实阴极电流。
- 阳极寄生电流。
- 物理测量电流。
- 仪器显示电流。
- 当前记录质量判定。

### P1：只调整截止模式记录解释与质量判定

在不新增 UI、不改变伏安模型的前提下，优先考虑：

1. 记录中继续保存当前外加反向电压作为截止电压。
2. 保留理论截止电压和偏差字段。
3. 保留或微调质量判定，使只有质量合格记录进入拟合。
4. 不阻止用户记录；即使质量较差也允许进入记录表，但不应默认作为可信拟合数据。

这与用户确认的：

```text
不限制记录，只记录当前电压；
拟合使用手动记录且质量合格的截止电压。
```

保持一致。

### P2：检查正常操作是否容易误导

如果实际验收中用户按照正常流程仍得到明显偏低的记录，应优先检查操作引导和默认状态，而不是替换电流公式。

重点包括：

1. 截止模式下用户看到的“测量电流”是否容易被误认为真实阴极电流。
2. 电流屏是否因调零基准和阳极寄生导致用户提前停在偏低电压。
3. 曲线区域是否已明确显示真实阴极电流和测量电流的区别。
4. 当前电压步进是否能让低截止电压档位精细调整。

### P3：必要时微调截止模式提示或默认流程

用户已确认原来没有的 UI 不新增，因此 P3 不做新面板。

但如果已有区域本来就显示相关信息，可以在不改变布局的前提下调整已有文字口径，例如：

- 把“电流”明确为“真实阴极电流”或“仪器读数”。
- 保持已有字段，但避免混用“测量电流”和“真实截止”。

若现有 UI 没有对应位置，本阶段不新增。

## 6. 不建议做的修改

根据用户确认方向，不建议做以下改动：

1. 不回退到物理框架大改之前。
2. 不恢复旧电流公式。
3. 不用旧指导文档中的高零偏收集率替换当前模型。
4. 不新增自动表观截止电压求解。
5. 不把 3D 电流屏改成真实阴极电流。
6. 不把暗电流、本底电流加入 3D 仪器读数。
7. 不新增 UI 分量面板或诊断面板。
8. 不把光强口径统一成百分比。
9. 不直接降低默认阳极寄生强度。
10. 不让伏安模式随 cutoff 修正一起重构。

## 7. 验收标准建议

后续若进入实现，应按以下标准验收。

### 7.1 电流口径

截止模式下应保持：

```text
真实阴极电流非负。
阳极寄生电流可为负。
物理测量电流是二者合计。
仪器显示电流再叠加调零基准。
```

### 7.2 记录口径

截止模式记录应保持：

```text
用户当前外加反向电压的绝对值作为手动记录截止电压。
理论截止电压作为对照。
质量合格记录进入拟合。
质量较差记录可保存但不应默认污染拟合。
```

### 7.3 正常操作结果

正常操作下，Cs 材料和五个滤光片的截止记录应接近理论值。

若出现明显系统性偏低，不应归咎于单个常量，而应综合检查：

- 操作流程。
- 调零状态。
- 阳极寄生影响。
- 真实阴极电流和测量电流显示差异。
- 记录质量判定。
- 拟合筛选逻辑。

## 8. 下一步建议

推荐下一步不是直接改代码，而是先做一份“当前模型标准场景数值核对表”：

```text
五个滤光片 × Cs × 4 mm × 40 cm × cutoff 模式
```

核对：

- 理论截止电压。
- 在理论截止点附近的真实阴极电流是否为 0。
- 阳极寄生电流有多大。
- 仪器显示电流如何变化。
- 当前质量判定是否会把正常操作记录判为 valid。

如果核对表显示问题来自记录判定，再改记录判定。

如果问题来自用户被仪器显示电流误导，再调整已有显示文字口径。

如果问题来自默认阳极寄生过强，再单独讨论是否改变默认实验设计。
