# 当前项目物理框架说明-new(26)

## 0. 文档定位

本文档整理的是当前项目已经落地的完整光电效应物理框架，而不是某一次近期改动的摘要。它覆盖从底层物理公式、实验状态、仪器读数、误差项、记录分析，到原理动画和 3D 场景显示之间的整体关系。

当前项目的核心原则是：

- 物理数值计算以 `src/features/experiment/photoelectricModel.js` 为统一内核。
- 实验仪器状态以 `src/stores/experiment.js` 为唯一可信状态源。
- 原理动画允许有独立演示状态，但复用同一个光电效应计算内核。
- 3D 微观小窗和 3D 仪器屏幕读取 experiment store，不再独立维护另一套电流公式。
- 正常阴极光电流只能为非负值；负电流必须来自明确的反向寄生项、仪器基准项或残余项。

## 1. 总体架构

### 1.1 分层结构

当前物理框架可以分为五层：

```text
物理常量与器材参数
  -> 光电效应数值模型
  -> experiment store 派生状态
  -> 2D/3D/动画/记录消费层
  -> 报告、诊断、拟合与展示
```

各层职责如下：

| 层级 | 主要文件 | 职责 |
|---|---|---|
| 常量层 | `src/features/experiment/constants.js` | 定义材料、滤光片、电流量程、测量模式、非包围式模型参数、电流预设、默认阳极污染参数 |
| 数值模型层 | `src/features/experiment/photoelectricModel.js` | 计算光子能量、最大初动能、截止电压、光通量、饱和电流、收集率、U95/U99、阳极寄生电流和能量一致运动 |
| 辅助层 | `src/features/experiment/helpers.js` | 构造默认状态、格式化仪表读数、处理调零、记录分析、工作流与诊断 |
| 状态层 | `src/stores/experiment.js` | 维护实验参数、仪器状态、记录数据，并给 2D/3D/报告提供统一 getter |
| 展示层 | `PrincipleAnimationSection.vue`、`PhotoelectricAnimationPanel.vue`、`ThreeDMicroAnimationPanel.vue`、`SplineScene.vue` | 把物理状态显示为控制面板、曲线、粒子动画、3D 仪器屏幕和实验场景 |

### 1.2 单一可信来源

项目中有两个状态域：

- `experiment store`：真实实验台、3D 场景、记录和报告的可信状态源。
- `PrincipleAnimationSection` 的局部状态：用于教学演示的独立控制源。

二者状态分离，但计算逻辑应保持一致：

```text
experiment store
  使用器材状态、光路状态、调零状态、阳极污染默认项
  面向 2D 实验台、3D 场景、仪器屏幕、记录和分析

PrincipleAnimationSection
  使用独立滑块与开关
  面向原理动画演示
  可演示暗电流、本底电流和阳极污染
```

这种分离的目的不是让两套物理逻辑分叉，而是让教学演示可以自由拖动参数，同时不破坏实验仪器流程。两边都应调用 `photoelectricModel.js` 中的核心公式。

## 4. 光电效应核心理论

### 4.1 光子能量

核心函数：

```js
resolvePhotonEnergyEv(frequencyHz)
```

公式：

```text
E_photon = h * nu
```

其中 `h` 使用 `PLANCK_EV`，所以结果单位是 eV。

### 4.2 最大初动能

核心函数：

```js
resolveMaxKineticEnergyEv({ frequencyHz, workFunctionEv })
```

公式：

```text
Kmax = max(0, E_photon - phi)
```

物理含义：

- `E_photon <= phi` 时，光子能量不足，无法产生正常光电子。
- `E_photon > phi` 时，电子最大初动能为多余能量。

### 4.3 理论截止电压

核心函数：

```js
resolveStoppingPotentialV(input)
```

公式：

```text
Vs = Kmax
```

因为 `Kmax` 以 eV 表示，数值上等于电子被同等伏特反向电压阻止时的电压值。

在 store 中，`currentCutoffVoltage` 也使用同一思想：

```text
Vs = max(0, PLANCK_EV * frequency - workFunction)
```

结论：

- 截止电压只由频率和材料逸出功决定。
- 光强、光阑、距离、仪器电流量程都不应改变理论截止电压。

## 5. 光通量和饱和电流

### 5.1 store 中的物理光通量

experiment store 使用光阑面积和距离平方反比：

```text
G = T * (D / D0)^2 * (d0 / d)^2
```

当前参考值：

```text
D0 = 4 mm
d0 = 40 cm
T = spectralTransmission，当前默认 1
```

因此在距离不变时，光阑 2/4/8 mm 的真实通量比例为：

```text
1 : 4 : 16
```

这套通量来自 `physicalLightFluxRatio`，它只在汞灯电源关闭时直接返回 0。盖子是否打开不在该 getter 中处理，而是在 `photoelectricConditionsReady` 中处理。

### 5.2 UI 光强显示值

`recalculateIntensity()` 会根据距离和光阑更新 `params.intensity`：

```text
intensity = 50 * (40 / distance)^2 * (aperture / 4)^2
```

然后限制到 `1~100`。

这个字段主要用于界面显示和原理动画输入。实验仪器电流计算不直接以这个 capped 百分比作为唯一通量来源，而是使用 `physicalLightFluxRatio`。

### 5.3 原理动画中的手动光强

原理动画没有真实光阑和距离控制，因此使用手动强度：

```text
normalizedFlux = intensity / 50
```

这意味着：

- 50% 对应参考通量 1。
- 100% 对应参考通量 2。
- 原理动画可以表达强弱趋势，但不是 3D 仪器光阑/距离模型的完整替代。

### 5.4 饱和电流

核心函数：

```js
resolveSaturationCurrentAmp({ normalizedFlux, quantumEfficiency, referenceCurrentAmp, currentLimitAmp })
```

线性饱和电流：

```text
I_s_linear = I_ref * QE * G
```

若预设没有电流上限：

```text
I_s = I_s_linear
```

若预设有仪器电流上限：

```text
I_s = I_limit * (1 - exp(-I_s_linear / I_limit))
```

当前两个光电流预设：

| key | 用途 | referenceCurrentAmp | currentLimitAmp |
|---|---|---:|---:|
| `normalizedTeaching` | 归一化教学演示 | `20e-13 A` | 无 |
| `zkyGd3Like` | ZKY-GD-3 论文数据近似 | `1.0e-10 A` | `3.5e-10 A` |

当前默认使用 `zkyGd3Like`。

## 6. 非包围式光电管收集率模型

### 6.1 为什么采用非包围式模型

当前项目不是把光电管视为“电子一逸出就几乎全部被收集”的包围式结构，而是采用非包围式近似。这样做是为了让伏安特性曲线在 0 V 附近不立即饱和，而是在正向电压升高后逐渐趋近饱和。

这更符合当前教学目标：

- 反向电压下可以观察逐渐遏止。
- 0 V 处只有一部分电子被收集。
- 正向电压增加会提高收集率。
- U95/U99 是收集率意义上的正向饱和电压，不是截止电压。

### 6.2 上升尺度 `Ur`

核心函数：

```js
resolveVoltAmpRiseVoltage()
```

当前公式：

```text
Ur = max(minRiseVoltage, referenceRiseVoltage * (Kmax / Kref)^frequencyExponent)
```

当前参数：

| 参数 | 当前值 |
|---|---:|
| `referenceFrequencyHz` | `5.49e14` |
| `referenceRiseVoltage` | `5.6 V` |
| `frequencyExponent` | `0.35` |
| `minRiseVoltage` | `3.5 V` |
| `positiveShapeExponent` | `1` |
| `retardingExponent` | `1.8` |
| `zeroBiasCollection` | `0.05` |

当前没有 `maxRiseVoltage` 主动上限。也就是说 U95/U99 不再因为人为封顶而停止变化。

需要注意：

- `Ur` 的变化源是 `Kmax`，也就是频率和材料逸出功。
- 光强不会进入 `Ur`。
- 光强只通过饱和电流改变纵向电流规模。

### 6.3 收集率 `C(U)`

核心函数：

```js
resolveNonEnclosingCollectionFraction()
```

当没有有效光电效应时：

```text
Vs <= 0 => C(U) = 0
```

当反向电压完全达到或超过截止条件时：

```text
U <= -Vs => C(U) = 0
```

当处于反向但尚未截止：

```text
C(U) = c0 * ((U + Vs) / Vs)^retardingExponent
```

当处于 0 V 或正向电压：

```text
C(U) = c0 + (1 - c0) * (1 - exp(-(U / Ur)^shape))
```

其中：

```text
c0 = zeroBiasCollection = 0.05
shape = positiveShapeExponent = 1
```

物理解释：

- `c0` 表示 0 V 附近仍有少量电子靠初动能和几何路径被收集。
- 反向区按剩余能量比例衰减。
- 正向区按指数形式趋近 1。

### 6.4 U95 和 U99

核心函数：

```js
resolveSaturationVoltage()
```

反解公式：

```text
U_ratio = Ur * [-ln((1 - ratio) / (1 - c0))]^(1 / shape)
```

当前展示：

| 指标 | 含义 |
|---|---|
| `U95` | 收集率达到 95% 时的正向电压 |
| `U99` | 收集率达到 99% 时的正向电压 |

关键结论：

- U95/U99 不是反向截止电压。
- U95/U99 不应随光强改变。
- U95/U99 当前没有人为最大上限。
- 若频率和材料不变，单独提高光强只会提高饱和电流，不会改变 U95/U99。

## 7. 电流口径体系

当前项目最容易混淆的是“真实阴极电流”“阳极寄生电流”“测量电流”和“仪器显示电流”。这四者必须分开。

### 7.1 正常阴极光电流

核心函数：

```js
resolveCathodePhotocurrentAmp()
```

公式：

```text
I_cathode = max(0, I_s * C(U))
```

含义：

- 这是由阴极材料吸收入射光子后释放的正常光电子形成的电流。
- 它永远不为负。
- 当未满足光电效应条件、光路未就绪或仪器关闭时，store 中的生成电流会为 0。

当前有两个收集率口径：

| 口径 | 含义 | 主要用途 |
|---|---|---|
| `baseCollectionFraction` | 非包围式光电管的基础收集率，达到理论截止条件时严格为 0 | 截止质量判定、判断基础截止门 |
| `collectionFraction` | 实际参与阴极电流计算的有效收集率 | 电流显示、3D 仪器、记录电流 |

在伏安模式下，两者相同。在截止电压模式下，若阳极寄生开启，项目会加入一段只用于截止模式的有效阴极尾流：

```text
C_eff(U) = C_base(U) + C_tail(U)
I_cathode = I_s * C_eff(U)
```

尾流量按当前阳极寄生参数在理论截止点 `U = -Vs` 处的负向贡献反推：

```text
C_tail0 = min(|I_anode_parasitic(U=-Vs)| / I_s, 0.02)
```

尾流在 `U < -Vs` 一侧不再使用固定宽度，而是按当前截止模式最低电压到 `-Vs` 的可达反向区间动态估算，最小宽度为 `0.12 V`，并按 smoothstep 曲线平滑衰减；在 `U >= -Vs` 一侧，尾流不再作为固定收集下限，而是按当前“无尾流表观零点”的距离估算右侧宽度并连续衰减。右侧局部补偿使用更慢释放的形状：

```text
t = clamp((U + Vs) / rightWidth, 0, 1)
C_tail_right(U) = C_tail0 * (1 - smoothstep(t)^2)
```

这样做的目的，是在不改阳极寄生电流、不改电流屏显示口径的前提下，让所有滤光片下的测量电流零点回到理论截止电压附近，同时避免某一固定窄区间承担全部补偿而导致电流示数突变；对 546 nm、577 nm 这类截止电压较小的滤光片，二次释放形状还可以减少尾流衰减和基础阴极电流上升互相抵消造成的零点附近平台感。

### 7.2 基础光电流噪声

store 中 `generatedPhotocurrentAmp` 在阴极模型电流上叠加一次基础噪声：

```text
I_generated = max(0, I_cathode_model * (1 + noise))
```

噪声幅度来自：

```text
BASE_PHOTOCURRENT_NOISE_RATIO = 0.05
```

即约 ±5%。该噪声会在频率、光强、材料，以及光路关键器件改变时重新采样。

### 7.3 阳极寄生电流

核心函数：

```js
resolveAnodeParasiticPhotocurrentAmp()
```

阳极寄生电流表示阳极污染和杂散光导致的反向寄生项。当前约束：

```text
U >= 0 => I_anode_parasitic = 0
```

反向电压下：

```text
I_anode_parasitic =
  -(I_forward_equivalent / K) * contaminationRatio * strayLightRatio
```

当前默认：

```text
K = 6
contaminationRatio = 0.52
strayLightRatio = 0.58
```

store 中的 `I_forward_equivalent` 不是随便取一个常数，而是在同一材料、同一频率、同一通量下，把电压取 `abs(U)` 后重新计算出的正向等效阴极电流。

物理含义：

- 正常阴极光电流来自阴极，方向按正常读数为非负。
- 阳极污染寄生电流来自阳极污染面或漏光影响，在反向条件下表现为负向贡献。
- 因此项目允许测量电流为负，但不允许正常阴极光电流为负。

### 7.4 暗电流和本底电流

当前项目对暗电流和本底电流的处理分为两种口径：

| 场景 | 暗电流 | 本底电流 |
|---|---|---|
| 原理动画截止电压模式 | 可开启并计入测量电流 | 可开启并计入测量电流 |
| 原理动画伏安模式 | 强制置 0 | 强制置 0 |
| experiment store / 3D 仪器 | 暂不计入，记录为 0 | 暂不计入，记录为 0 |

原理动画中的最大值：

```text
BACKGROUND_CURRENT_MAX_AMP = 1.8e-13
DARK_CURRENT_MAX_AMP = 1.1e-13
```

当前实现意图是：3D 仪器主要考虑阳极寄生电流，暗电流和本底电流影响较小，暂不纳入 3D 仪器读数。

### 7.5 测量电流

在原理动画中：

```text
I_measured =
  I_true_cathode
  + I_anode_parasitic
  + I_dark
  + I_background
```

在 experiment store / 3D 仪器中：

```text
I_measured =
  I_generated_cathode
  + I_anode_parasitic
```

其中：

- `I_generated_cathode` 是加入基础噪声后的正常阴极电流。
- 暗电流和本底电流当前为 0。

### 7.6 仪器基准电流

仪器基准电流来自初始零漂和调零旋钮：

```text
I_reference =
  I_instrument_bias
  + zeroAdjust * 1e-12 A
```

初始零漂范围：

```text
-1e-12 A ~ -1e-13 A
```

调零旋钮范围：

```text
zeroAdjust = -50 ~ 50
currentAmpPerUnit = 1e-12 A
```

### 7.7 仪器显示电流

store getter：

```js
currentPhotocurrentAmp()
```

当前逻辑：

```text
仪器关闭:
  I_meter_raw = 0

后方电流线未接入:
  I_meter_raw = I_reference

后方电流线已接入:
  I_meter_raw = I_measured + I_reference
```

3D 左侧电流屏和集成控制台显示的是：

```text
currentMeterDisplayText = formatCurrentMeterDisplay(I_meter_raw, currentRange)
```

所以 3D 电流显示屏显示的是“仪器测量读数”，不是单独的真实阴极电流。

### 7.8 记录电流

`recordData()` 中的 `current` 字段取自仪器显示电流口径，并叠加一次记录测量噪声：

```text
record.current =
  currentPhotocurrentAmp * (1 + measurementNoiseRatio)
```

测量噪声范围：

```text
-0.03 ~ 0.03
```

记录中的 `measuredPhotocurrentAmp` 不混入调零基准，也不叠加这一次记录噪声；它保留物理测量电流口径。

## 8. 调零和测量流程

### 8.1 就绪条件

`photoelectricConditionsReady` 表示光电效应条件是否就绪：

```text
lampPowerOn
photocellCoverOpen
lampCoverOpen
intensity > 0
currentCutoffVoltage > 0
```

`isReadyToMeasure` 表示是否可以记录：

```text
instrumentPowerOn
lampPowerOn
photocellCoverOpen
lampCoverOpen
rearCableConnected
zeroConfirmed
```

二者不同：

- `photoelectricConditionsReady` 关注是否能发生光电效应。
- `isReadyToMeasure` 关注仪器流程是否允许采集数据。

### 8.2 调零逻辑

当前调零流程：

1. 仪器电源开启后存在初始零漂。
2. 用户通过 `zeroAdjust` 把 `referenceCurrentAmp` 调到零附近。
3. 后方电流线接入前，如果基准电流不接近 0，会阻止接线。
4. 接线后才能确认调零点。
5. 确认后记录 `zeroReferenceCurrent`。

零点判定：

```text
ZERO_CURRENT_TOLERANCE_AMP = 1e-13
```

记录时的零点校验：

- 若参考电流绝对值小于 `1e-13 A`，使用绝对容差 `1e-13 A`。
- 否则使用相对容差 `5%`。

### 8.3 电压和测量模式

测量模式定义：

| 模式 | key | 电压范围 | 用途 |
|---|---|---|---|
| 截止电压测试 | `cutoff` | `-2 V ~ 2 V` | 手动找反向截止点 |
| 伏安特性测试 | `voltAmpere` | `-2 V ~ 30 V` | 扫描 I-U 曲线 |

切换测量模式会同步切换电压范围，并把当前电压夹到对应范围内。

## 10. 截止电压模式

### 10.1 物理目标

截止电压模式的目标是找到反向电压使正常阴极光电子无法被阳极收集：

```text
U_applied ≈ -Vs
```

记录中将手动截止电压写作：

```text
cutoffVoltage = max(0, -U_applied)
```

### 10.2 截止质量判定

当前质量判定函数使用基础截止门收集率，而不是截止模式有效尾流后的收集率：

```text
cutoffGateCollectionFraction <= 0.005
```

并比较：

```text
cutoffVoltageDeviation = cutoffVoltage - theoreticalCutoffVoltage
```

质量等级：

| 等级 | 条件 |
|---|---|
| `valid` | 收集率足够接近 0，且偏差不超过 0.08 V |
| `suspect` | 收集率接近 0 或偏差不超过 0.18 V |
| `invalid` | 不满足以上条件 |

只有 `valid` 的截止模式记录会进入拟合数据集。

这样处理是为了避免 A 路径的有效尾流把“电流补偿量”误认为“尚未截止”。在 `U=-Vs` 附近，基础截止门可以为 0，同时有效阴极电流仍可用来抵消阳极寄生，使测量电流接近 0。

### 10.3 截止模式中的电流显示口径

原理动画的曲线区域在截止模式下显示：

| 行 | 内容 |
|---|---|
| 第一行 | 真实阴极电流 |
| 第二行 | 阳极电流 |
| 第三行 | 测量电流 |

这里的“阳极电流”用于单独观察阳极寄生项的方向和量级；“测量电流”用于体现阳极寄生电流、暗电流和本底电流可能导致的残余读数；“真实阴极电流”用于判断真正的光电截止。

## 11. 伏安特性模式

### 11.1 物理目标

伏安特性模式的目标是观察同一材料、频率、光通量下，电流随外加电压变化的曲线：

```text
I(U) = I_s * C(U)
```

其中：

- 反向区表现为逐渐遏止。
- 0 V 处有 `c0` 级别收集率。
- 正向区随电压升高趋近饱和。

### 11.2 自动扫描

`startAutoScan()` 会：

1. 切换到 `voltAmpere` 模式。
2. 设置电压范围为 `-2 V ~ 30 V`。
3. 清空记录。
4. 从 `-2 V` 到 `30 V`，每 `0.5 V` 记录一条。
5. 每条记录复用 `recordData()`。

### 11.3 伏安模式中的显示口径

原理动画曲线区域在伏安模式下显示：

| 行 | 内容 |
|---|---|
| 第一行 | 电流 |
| 第二行 | 电压 |
| 第三行 | 收集率 |

伏安模式下，原理动画会把暗电流和本底电流置 0，只保留真实阴极电流和阳极寄生项的影响。

## 12. 电流曲线和 UI 归一化

`PhotoelectricAnimationPanel.vue` 负责电流曲线区域。

### 12.1 曲线输入

它先调用：

```js

```

得到：

- `trueCathodePhotocurrentAmp`
- `parasiticAnodePhotocurrentAmp`
- `darkCurrentAmp`
- `backgroundCurrentAmp`
- `measuredPhotocurrentAmp`
- `collectorSuccessRatio`
- `saturationCurrentAmp`
- `voltageReferenceMax`

### 12.2 数据条基准

当前数据条使用统一基准：

```text
currentScaleReferenceAmp =
  max(
    saturationCurrentAmp,
    trueCathodePhotocurrentAmp,
    measuredPhotocurrentAmp,
    photocurrentAmp,
    parasiticAnodePhotocurrentAmp,
    1e-13
  )
```

然后：

```text
真实阴极电流条 = abs(I_true) / currentScaleReferenceAmp
测量电流条 = I_measured / currentScaleReferenceAmp
```

这样可以保证：

- 各数据条使用同一长度基准。
- 左端对齐。
- 同一组数据在调整电压时不会因为单条基准变化而视觉跳动。
- 负向测量电流可以使用负向色调，但条长仍按绝对值显示。

### 12.3 文本布局

当前曲线面板中的标签和数值为单行展开，不再分成两行。这是 UI 层约束，不改变物理计算。

## 14. 动画物理映射

### 14.1 映射入口

`src/features/photoelectric-animation/mapping.js` 中的：

```js
resolvePhotoelectricAnimationState(input)
```

把数值物理量映射为动画状态。

主要输入：

- 频率、光强、电压、材料、逸出功。
- 灯、盖子和光路状态。
- 真实阴极电流。
- 阳极寄生电流。
- 暗电流和本底电流。
- 测量电流。
- 动画模式。

主要输出：

- 光子能量、逸出功、阈频、最大初动能。
- 场景状态：未入射、低于阈频、遏止、截止、收集。
- 电场方向、电场强度、电场力、加速度。
- 真实电流、寄生电流、残余电流、测量电流的归一化值。
- 粒子发射率、收集率、低能背景粒子速率。
- U95/U99、饱和电流、收集率等模型派生值。

### 14.2 场景判定

当前场景大致按以下顺序判定：

```text
无灯光或光路关闭
  -> light-off

有光但 photonEnergy <= workFunction
  -> below-threshold

可发射，且测量/真实电流接近截止阈值
  -> cutoff-near

反向电压但未截止
  -> retarding

0 V 或正向
  -> emission / accelerating
```

### 14.3 数值模型与视觉模型的边界

动画状态既包含真实模型值，也包含视觉辅助值：

| 类型 | 示例 | 用途 |
|---|---|---|
| 数值模型值 | `trueCathodePhotocurrentAmp`、`measuredPhotocurrentAmp`、`cathodeCollectionFraction` | 与曲线、标签和模型读数对齐 |
| 视觉映射值 | `photonRate`、`escapeProbability`、`scatterLossRatio`、`currentLevel` | 控制粒子数量、速度、散射和视觉密度 |
| 场景解释值 | `stageHeadline`、`parameterNotes`、`statusTags` | 给动画面板显示现象解释 |

因此，动画中的粒子数量不是直接等于电子数，而是按物理趋势映射出的教学可视化密度。

## 15. 电子运动和伏安轨迹

### 15.1 简化电场力映射

`src/features/photoelectric-animation/physics.js` 提供简化动画力学：

```text
electricFieldVoltPerPx = voltage / plateDistancePx
```

电子受力方向：

```text
voltage > 0 => 电子向阳极方向加速
voltage < 0 => 电子受反向遏止
```

动画加速度：

```text
animationAccelerationPxPerS2 =
  direction * strengthRatio * FIELD_TO_ANIMATION_ACCELERATION
```

这不是完整 SI 单位仿真，而是把电场强弱投影到屏幕空间。

### 15.2 能量一致运动

`photoelectricModel.js` 中的 `resolveEnergyConsistentMotion()` 用动能决定初速度：

```text
speed_px = speedScale * sqrt(K_sample)
```

电压对应 x 方向加速度：

```text
a_px = sign(U) * speedScale^2 * abs(U) / (2 * plateDistancePx)
```

这使得：

- 动能越高，电子初速度越大。
- 反向电压越强，电子越容易减速返回。
- 正向电压越强，电子越容易快速到达阳极。

### 15.3 伏安轨迹采样

`voltAmpTrajectory.js` 会采样多组：

- 发射角度。
- 初动能比例。
- 阴极表面出射位置。

然后判断电子是否：

- 被反向场拉回阴极。
- 飞出场区。
- 错过阳极收集窗口。
- 成功到达阳极。

采样结果给出：

```text
collectionRatio = collectedSamples / totalSamples
```

不过当前伏安模式最终会优先使用 `cathodeModel.collectionFraction`，轨迹采样更多用于动画运动解释和视觉一致性。

## 18. 报告与拟合

### 18.1 拟合数据来源

`buildFitRecords()` 只使用：

- 截止模式记录。
- `cutoffQuality === valid`。
- 有有效 `cutoffVoltage` 的记录。

同一滤光片多次记录时，保留最新一条。

### 18.2 线性回归

拟合关系：

```text
U_c = (h/e) * nu - W0/e
```

代码中 `buildRegression()` 对：

```text
x = frequency
y = cutoffVoltage
```

做线性回归，得到：

```text
slope
intercept
```

换算：

```text
planckConstant = slope * ELEMENTARY_CHARGE
workFunction = abs(intercept)
thresholdFrequency = workFunction / PLANCK_EV
```

### 18.3 可分析条件

至少需要两种不同频率的有效截止电压记录，才会给出普朗克常量估计。

## 19. 当前假设、限制和边界

### 19.1 物理模型限制

- 非包围式收集率是教学/实验复现近似，不是完整电极几何和空间电荷求解。
- 阳极寄生电流使用等效正向阴极电流缩放，不是单独求解阳极污染面的材料逸出功。
- 基础光电流噪声和记录噪声是经验随机项，不代表真实仪器所有噪声来源。
- 原理动画粒子数量是视觉密度映射，不是电子数的逐个真实统计。

### 19.2 当前明确口径

- 默认阴极材料为 Ag-O-K。
- 理论截止电压只由频率和材料决定。
- 光强只影响饱和电流，不影响截止电压、U95、U99。
- U95/U99 当前没有人为最大上限。
- 正常阴极光电流不允许为负。
- 负测量电流主要来自阳极寄生电流或仪器基准项。
- 3D 仪器读数计入阳极寄生电流和调零基准。
- 3D 仪器读数暂不计入暗电流和本底电流。
- 原理动画可演示暗电流和本底电流，但伏安模式下会置 0。

### 19.3 容易混淆的名词

| 名词 | 正确含义 |
|---|---|
| 截止电压 `Vs` | 反向阻止最大初动能电子到达阳极所需电压 |
| U95/U99 | 正向电压下达到 95%/99% 收集率的饱和指标 |
| 真实阴极电流 | 正常阴极光电效应产生的非负电流 |
| 阳极寄生电流 | 阳极污染和漏光造成的反向负向贡献 |
| 测量电流 | 各物理电流项合成后的测量口径，不含调零基准 |
| 仪器显示电流 | 测量电流加上仪器基准后按量程格式化的读数 |
| 记录 `current` | 仪器显示电流口径再叠加一次记录噪声 |

## 20. 维护规则

后续调整底层物理框架时建议遵守以下规则：

1. 优先修改 `photoelectricModel.js`，再同步 store、原理动画和 3D 映射。
2. 不要让正常阴极光电流出现负值。
3. 不要把阳极寄生电流、暗电流、本底电流混同为正常阴极电流。
4. 不要把正向饱和电压 U95/U99 与反向截止电压 Vs 混用。
5. 不要把光强引入截止电压或 U95/U99，除非有明确的新物理模型依据。
6. 若新增 3D 残余电流来源，需要同时更新：
   - experiment store getter
   - 3D 微观小窗输入
   - 3D 仪器屏读数说明
   - 实验记录字段
   - 单元测试
   - 本说明文档
7. 若调整电流显示口径，需要同时核对：
   - `measuredPhotocurrentAmp`
   - `currentPhotocurrentAmp`
   - `currentMeterDisplayText`
   - `record.current`
   - 电流曲线区域标签
8. 若调整伏安曲线模型，需要同步验证：
   - `collectionFraction`
   - `saturationCurrentAmp`
   - `voltAmpRiseVoltage`
   - `saturationVoltage95`
   - `saturationVoltage99`
   - 自动扫描记录
   - 原理动画粒子收集率

## 21. 快速核对清单

修改后至少检查：

- 材料切换是否正确改变逸出功、阈频和截止电压。
- 光强变化是否只改变电流规模，不改变理论截止电压。
- 频率变化是否改变截止电压、U95/U99 和最大初动能。
- 反向电压到达 `-Vs` 附近时，真实阴极电流是否接近 0。
- 阳极污染开启且反向电压下，测量电流是否可能小于真实阴极电流。
- 3D 电流屏是否显示仪器读数，而不是单独真实阴极电流。
- 原理动画和 3D 微观小窗是否使用同一底层模型。
- 记录字段是否能区分真实阴极电流、寄生电流、测量电流和仪器读数。
- 普朗克常量拟合是否只使用有效截止电压记录。
