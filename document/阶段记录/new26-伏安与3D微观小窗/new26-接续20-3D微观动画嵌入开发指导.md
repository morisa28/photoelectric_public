# 阶段接续文档20-new(26)-3D微观动画嵌入开发指导

生成时间：2026-06-03  
适用分支：`codex/principle-volt-amp-mode-continue`  
适用范围：`new(26)` 光电效应实验模拟仿真平台的 3D 页面微观动画嵌入改造。

## 1. 总目标

将原理动画区域中已经成熟的微观动画表达能力，缩减后嵌入 `/3d-view` 的微观动画小窗，替代当前 3D 页原有的普通微观小窗展示。改造完成后，3D 小窗应直接服务实验台实际参数，随 3D 模型中的测试模式、电压、光强、频率、材料、开盖和电源状态变化而同步更新。

本任务的核心不是复制原理动画区组件，而是在现有 `PhotoelectricAnimationPanel` 复用层上增加 3D 专用适配，使状态边界清晰、视觉密度适合 3D 小窗，并保留现有原理动画区的独立教学演示能力。

- 原理动画区入口：`src/components/PrincipleAnimationSection.vue`。

- 项目内 `AGENTS.md` 明确：普通开发默认不创建新的 Git worktree，本任务在当前工作区按小阶段提交。

## 3. 状态架构决策

### 3.1 原理动画区状态

原理动画区继续使用 `PrincipleAnimationSection.vue` 内部的独立状态。截止电压模式与伏安特性模式各自保存一份控制参数，不与 3D 实验台共享，也不受 3D 模型按钮影响。

该边界用于保证原理动画区仍是教学演示区，可自由调参数，不污染正式实验数据。

### 3.2 3D 微观小窗状态

3D 小窗必须从 `useExperimentStore()` 派生全部实验参数。3D 内部的截止电压模式与伏安特性模式共用同一套 `params`、`apparatus`、当前电流、电压量程和仪表状态，不创建模式私有副本。

映射原则：

| 3D store 字段 | 微观动画字段 |
| --- | --- |
| `apparatus.measurementMode === 'cutoff'` | `animationMode = 'cutoff-voltage'` |
| `apparatus.measurementMode === 'voltAmpere'` | `animationMode = 'volt-amp'` |
| `params.voltage` | `voltage` |
| `params.frequency` | `frequency` |
| `params.intensity` | `intensity` |
| `params.material` | `material` |
| `currentCutoffVoltage` | `cutoffVoltage` |
| `generatedPhotocurrentAmp` / `currentPhotocurrentAmp` | 正常光电流或测量电流 |
| `apparatus.lampPowerOn`、盖子状态 | 入射光与光路状态 |

## 4. 组件设计方案

新增 3D 专用适配组件，建议命名为：

- `src/components/animation/ThreeDMicroAnimationPanel.vue`

职责：

- 只读取 `experimentStore`，不持有独立实验参数。
- 负责把实验 store 映射成 `PhotoelectricAnimationPanel` props。
- 固定开启阳极污染演示。
- 固定关闭本底电流、暗电流及其独立电子源。
- 启用紧凑视觉模式。
- 不提供原理区的完整控制栏。

`PhotoelectricAnimationPanel.vue` 继续作为可复用展示面板。为避免影响原理动画区，应新增可选 prop，例如：

- `compact`：启用 3D 小窗紧凑 DOM 样式。
- `canvasLabelScale`：传入 Canvas 标签字号倍率。
- `legendDense` 或直接由 `compact` 控制图例间距和尺寸。

`engine.js` 不应继续只使用全局固定标签缩放。应允许 `animationState.labelScale` 或同类字段影响绘制，但默认值保持当前原理动画效果不变。3D 小窗传入 `0.75`，表示文字标签缩小到当前的 75%。

## 5. 3D 小窗视觉缩减要求

3D 小窗默认行为：

- 阳极污染演示常态开启。
- 不显示本底电流和暗电流电子。
- 图例包含：光子、电子、势垒、能量条、电场、阳极、电流曲线、阳极污染区、阳极电子。
- 图例字号、标记尺寸、间距按当前 75% 附近缩小。
- Canvas 内区域标签、入射光照标签、电流/电压/收集率等内部标签按当前 75% 缩小。
- 小窗默认打开与否沿用当前按钮逻辑，不强制每次进入 3D 自动展开，避免遮挡 3D 模型。

## 6. 可缩放边界方案

当前 `ThreeDView.vue` 只有拖动位置，没有尺寸状态。应新增：

- `animationPanelSize = ref({ width, height })`
- `activePanelResize` 运行态对象
- `animationPanelStyle` 同时输出 `left/top/bottom/width/height`
- 四边或八向 resize handle

交互原则：

- 用户可拖动左右边界改变宽度。
- 用户可拖动上下边界改变高度。
- 四角同时改变宽高，便于真实使用。
- 尺寸 clamp 到视口内，避免小窗拖到不可操作区域。
- 最小尺寸建议：`420 x 360`，移动端可更低。
- 最大尺寸建议：视口宽高减去安全边距。
- resize 与 drag 都使用 pointer 事件，释放时清理监听器。
- Canvas 已有 `ResizeObserver`，容器尺寸变化后应自动重绘。

## 7. 测试模式按钮与量程联动

`SplineScene.vue` 中 `Btn_Query001` 与 `Btn_VoltAmpTest` 已映射到 `toggle-measurement-mode`，当前会调用 `experimentStore.toggleMeasurementMode({ source: '3d' })`。

本次开发保持该链路为唯一入口：

- 按下测试模式按钮后切换 `apparatus.measurementMode`。
- 微观动画模式由 `apparatus.measurementMode` 派生，不单独切换。
- 电压表显示和量程由 `experimentStore.setMeasurementMode/toggleMeasurementMode` 内的 `voltageRange`、`currentVoltageBounds` 和 `voltageDisplayText` 派生。

用户原始描述中“微观动画逻辑切换为截止电压模式”与“电压表显示量程变更为符合伏安特性测量”存在语义交叉。本阶段按工程一致性执行：当前实际测量模式是什么，微观动画和电压量程就同步到同一模式。若后续视觉验收要求按钮含义反向，再只调整模式映射，不改状态架构。

## 10. 风险与处理

- 当前工作区已有前序未提交改动，后续开发前必须提交为基线，否则阶段回退会混乱。
- 3D 小窗缩小后，图例和电流统计区可能拥挤；优先通过紧凑模式和可缩放窗口解决，而不是删除必需图例。
- Canvas 标签缩放若做成全局常量会影响原理动画区，必须通过状态/prop 控制。
- 测试模式按钮的用户语义存在歧义，本阶段按实验实际模式一致性实现。
- 3D 页面叠加小窗可能遮挡 toolbar 或 Spline 操作，resize/drag clamp 必须保留安全边距。
