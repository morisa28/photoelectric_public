## 1. 任务目标

本阶段要完成 6 类修正：

1. 原理动画区的截止电压模式和伏安特性模式，初始外加电压统一改为 `-1 V`。
2. 伏安特性模式下，电子轨迹点生命周期从当前 `4 s` 缩短为 `1.5 s`。
3. 重写动画电场线表达：
   - 电场线方向应与电子受力方向相反。
   - 电场线密度随外加电压绝对值线性变化。
   - `0 V` 时隐藏电场线，但仍保留间距计算结果。
4. 重写原理动画区讲解界面：
   - 点击讲解按钮后弹出非阻塞浮窗。
   - 浮窗可自由拖动。
   - 浮窗存在时，用户仍可操作参数并观看动画。
   - 讲解文本中的参数和读数应随用户调节即时更新。
5. 重写收集率计算逻辑：
   - 每 5 秒统计一次击中阳极的正常电子占全部正常电子的比例。
   - 文档和界面应明确当前电流计算逻辑，以及电流与当前收集率是否直接关联。
6. 修复模式切换即时刷新问题：
   - 原理动画区切换截止电压/伏安特性时，动画逻辑当下更新。
   - 3D 微观小窗随 3D 测试模式切换时，动画逻辑当下更新。

### 2.4 当前 Canvas 刷新问题

`PhotoelectricAnimationCanvas.vue` 的 `animationStateKey` 当前包含 scene、presentation、focus、电压、频率、电流等字段，但未包含 `animationMode`。因此当模式切换但电压等连续数值没有变化时，watcher 不触发，必须调一次滑块才会更新引擎状态。

结论：

- 第 6 项的核心修复点是 `PhotoelectricAnimationCanvas.vue`。
- 只要补齐 key，原理动画区和 3D 微观小窗会同时受益。

### 2.5 当前电场线绘制

`engine.js` 中 `drawField()` 当前行为：

- 使用 `resolvePhotoElectronFieldAcceleration(animationState)` 得到电子受力/加速度方向。
- 箭头方向与电子加速度方向一致。
- 电场线间距固定为 `34 px`。
- `0 V` 时只要满足 lamp/canEmit 条件，仍可能绘制弱电场线。

结论：

- 需要把“电子受力方向”和“电场线方向”分离。
- 不能反转电子运动加速度，否则会破坏截止/伏安已有运动逻辑。
- 电场线密度规则应抽成纯函数，便于单元测试。

### 2.6 当前收集率逻辑

`engine.js` 当前已有：

```js
const COLLECTION_RATIO_WINDOW_SECONDS = 5
let normalCollectionSamples = []
```

当前收集率是近 5 秒滚动窗口即时计算：

```js
const collected = normalCollectionSamples.filter((sample) => sample.collectedAt != null).length
return clamp(collected / normalCollectionSamples.length, 0, 1)
```

结论：

- 这不是“每 5 秒统计一次并保持显示”的结算制。
- 需要改成 interval bucket。
- UI 文案应从“实时收集率”调整为“5 秒收集率”或“周期收集率”，避免显示语义不一致。

## 3. 设计原则

### 3.1 不混淆教学状态和正式实验状态

原理动画区是教学演示状态，3D 微观小窗是真实实验状态适配。二者共享动画引擎，但状态来源不同。本阶段不得把原理动画的本地参数写入 `experimentStore`，也不得让 3D 小窗持有原理动画模式副本。

### 3.2 不用视觉规则反推物理运动

电场线方向和密度是视觉表达；电子运动由现有 `electronFieldAcceleration`、`voltAmpMotion.accelerationPxPerS2` 和 `stepElectronFlight()` 决定。实现时只应改变 `drawField()` 的箭头和间距，不应反转 `stepElectronFlight()` 的加速度符号。

### 3.3 先抽纯函数，再接 Canvas

电场线密度、线位置、5 秒收集率结算都适合抽成可测试函数或小型状态机。不要把规则全部写在 `drawField()` 或 `step()` 里，否则后续很难验收边界条件。

### 3.4 先修刷新链路，再调视觉

模式切换即时刷新是多个需求的前置依赖。若先改电场线和轨迹，Canvas 仍不刷新，会导致验收时误判实现无效。

### 3.5 小阶段交付硬性约束

每个小阶段必须按下面顺序收口：

1. 完成该阶段限定范围内的代码或文档改动。
2. 运行该阶段对应的最小验证命令。
3. 在 `document/` 下生成阶段总结文档，记录完成内容、改动文件、验证结果、未解决风险和下一阶段建议。
4. 执行 `git status --short` 和 `git diff --stat`，确认只包含本阶段相关改动。
5. 提交 Git commit，提交信息应与本阶段目标对应。

禁止把多个小阶段的改动长期堆在同一个未提交工作区中。若某阶段验证命令无法运行，也必须在阶段总结文档中写明原因、影响和后续补验方式，再提交该阶段结果。

#### 4.4.2 间距规则

新增纯函数：

```js
export function resolveFieldLineSpacingUnit({ animationMode, voltage }) {
  const absVoltage = Math.abs(Number(voltage) || 0)
  const isVoltAmp = animationMode === 'volt-amp'
  const maxVoltage = isVoltAmp ? 30 : 2
  const maxSpacing = 2
  const minSpacing = isVoltAmp ? 0.8 : 1.75
  const ratio = clamp(absVoltage / maxVoltage, 0, 1)

  return {
    visible: absVoltage > 0.001,
    spacingUnit: lerp(maxSpacing, minSpacing, ratio),
  }
}
```

规则解释：

- 截止电压模式：
  - `|V| = 0`：间距单位 `2`，但不显示电场线。
  - `|V| = 2`：间距单位 `1.75`。
  - 中间线性插值。
- 伏安特性模式：
  - `|V| = 0`：间距单位 `2`，但不显示电场线。
  - `|V| = 30`：间距单位 `0.8`。
  - 中间线性插值。
- 符号只决定方向，不决定间距。

#### 4.4.3 线位置规则

新增纯函数：

```js
export function resolveFieldLineYs({ top, bottom, spacingPx }) {
  const center = (top + bottom) / 2
  const ys = [center]

  for (let y = center - spacingPx; y >= top - spacingPx; y -= spacingPx) {
    ys.unshift(y)
  }

  for (let y = center + spacingPx; y <= bottom + spacingPx; y += spacingPx) {
    ys.push(y)
  }

  return ys
}
```

绘制时使用 clip：

```js
ctx.save()
ctx.beginPath()
ctx.rect(surface, top, collectorX - surface, bottom - top)
ctx.clip()
// draw lines
ctx.restore()
```

这样满足：

- 间距过大导致部分线越界时，越界部分隐藏。
- 间距过小导致边界到最近线距离超过一个间距时，继续补线。
- 电场线始终只显示在真空飞行区内。

#### 4.4.4 drawField 改造

核心替换思路：

```js
const acceleration = resolvePhotoElectronFieldAcceleration(animationState)
const fieldDirection = -Math.sign(acceleration)
const { visible, spacingUnit } = resolveFieldLineSpacingUnit({
  animationMode: animationState.animationMode,
  voltage: animationState.voltage,
})

if (!visible || fieldDirection === 0) {
  return
}

const spacingPx = 34 * spacingUnit
const lineYs = resolveFieldLineYs({
  top: getFieldTop(),
  bottom: getFieldBottom(),
  spacingPx,
})
```

箭头：

- `fieldDirection > 0`：箭头向右。
- `fieldDirection < 0`：箭头向左。
- 电子受力方向与其相反。

测试建议：

- `resolveFieldLineSpacingUnit({ animationMode: 'cutoff-voltage', voltage: 0 })` 返回 `visible=false`、`spacingUnit=2`。
- 截止模式 `voltage=2` 或 `-2` 返回 `1.75`。
- 伏安模式 `voltage=30` 或 `-30` 返回 `0.8`。
- 中间值线性，例如伏安 `15 V` 返回 `1.4`。
- 电场方向与 `electronAcceleration.animationAccelerationPxPerS2` 符号相反。

#### 4.6.1 定义正常电子口径

本阶段“正常电子”建议定义为：

- `event.sourceKind === 'photo'`
- 成功逸出并进入飞行阶段
- 已执行 `registerNormalElectronEmission(event)`

排除：

- 阳极污染电子 `parasiticEvents`
- 本底电子 `sourceKind === 'background'`
- 暗电子 `sourceKind === 'dark'`
- 低能失败电子

#### 4.6.2 新状态

替换或补充当前滚动窗口：

```js
let normalIntervalStartedAt = 0
let normalIntervalEmitted = 0
let normalIntervalCollected = 0
let settledNormalCollectionRatio = 0
```

reset 时清零：

```js
normalIntervalStartedAt = 0
normalIntervalEmitted = 0
normalIntervalCollected = 0
settledNormalCollectionRatio = 0
```

发射时：

```js
function registerNormalElectronEmission(event) {
  ...
  normalIntervalEmitted += 1
}
```

收集时：

```js
function registerNormalElectronCollection(event) {
  ...
  normalIntervalCollected += 1
}
```

每帧结算：

```js
function updateNormalCollectionInterval() {
  if (normalIntervalStartedAt === 0) {
    normalIntervalStartedAt = metricClock
  }

  if (metricClock - normalIntervalStartedAt < COLLECTION_RATIO_WINDOW_SECONDS) {
    return
  }

  settledNormalCollectionRatio = normalIntervalEmitted > 0
    ? clamp(normalIntervalCollected / normalIntervalEmitted, 0, 1)
    : 0

  normalIntervalStartedAt = metricClock
  normalIntervalEmitted = 0
  normalIntervalCollected = 0
}
```

在 `step()` 中调用：

```js
updateNormalCollectionInterval()
```

快照：

```js
liveNormalCollectionRatio: settledNormalCollectionRatio
```

#### 4.6.4 电流与收集率关系说明

当前电流计算逻辑：

1. 原理动画区真实阴极光电流：

```js
const saturationCurrent = Math.max(
  PHOTOCURRENT_SATURATION_CURRENT_MIN_AMP,
  (animationControls.value.intensity / 100) * PHOTOCURRENT_SATURATION_CURRENT_MAX_AMP,
)
const current = saturationCurrent * Math.tanh((animationControls.value.voltage + currentCutoffVoltage.value) / 0.45)
```

2. 原理动画区测量电流：

```js
measured = trueCathodePhotocurrentAmp
  + parasiticAnodePhotocurrentAmp
  + effectiveBackgroundCurrentAmp
  + effectiveDarkCurrentAmp
```

3. 3D 微观小窗真实阴极光电流来自 `experimentStore.generatedPhotocurrentAmp`，公式同样是 `tanh((voltage + cutoff) / 0.45)`，并带基础噪声。

4. 动画引擎内部电流曲线：

- 正常光电流使用外部传入的 `normalCurrentSignedLevel`。
- 暗电流、本底电流会按各自实际收集比例折算。
- 当前正常电子收集率不反向参与真实阴极光电流计算。

结论：

- 当前电流和收集率趋势相关，但不是同一个计算源。
- 电流是仪表/物理响应模型输入。
- 收集率是动画粒子统计输出。
- 本阶段若只按需求重写收集率，不应把电流公式改为 `电流 * 收集率`，否则会改变仪表读数和伏安曲线行为。

## 6. 详细验收标准

### 6.1 初始电压

- 原理动画区首次进入截止电压模式：外加电压为 `-1.000 V`。
- 首次切到伏安特性模式：外加电压为 `-1.000 V`。
- 两模式仍各自保存独立参数状态。

### 6.2 轨迹生命周期

- 伏安特性轨迹点约 `1.5 s` 后消失。
- 轨迹仍从发射点连续到电子当前位置。
- 归档轨迹和实时轨迹都按点年龄淡出。

### 6.3 电场线

- 电压为 `0 V` 时不显示电场线。
- 正电压和负电压时，电场线箭头与电子运动受力方向相反。
- 截止模式：
  - `|V| = 0` 的间距单位为 `2`。
  - `|V| = 2` 的间距单位为 `1.75`。
- 伏安模式：
  - `|V| = 0` 的间距单位为 `2`。
  - `|V| = 30` 的间距单位为 `0.8`。
- 中间电压线性插值。
- 线条不画出真空飞行区。
- 边界到最近电场线距离不超过一个当前间距。

### 6.4 讲解浮窗

- 点击讲解按钮弹出浮窗。
- 浮窗可拖动。
- 浮窗打开时仍可操作频率、光强、电压、材料、污染等控件。
- 控件变化后，浮窗中的相关数值即时更新。
- 浮窗关闭后不残留 window 事件监听。

### 6.5 收集率

- 收集率每 5 秒结算一次。
- 统计口径只包括正常光电子。
- 阳极污染电子、本底电子、暗电子不进入正常电子收集率。
- UI 文案明确是 5 秒统计口径。
- 文档明确当前电流不由该收集率直接重算。

### 6.6 模式切换

- 原理动画区点击模式按钮后，Canvas 当下切换逻辑。
- 3D 微观小窗随 `Btn_Query001` / `Btn_VoltAmpTest` 切换后，当下切换逻辑。
- 不需要再调一次滑块才能看到逻辑变化。

## 7. 风险和注意事项

1. 不要把 `electricFieldDirection` 误用于 `stepElectronFlight()`。
2. 不要把原理动画默认电压改动扩散到正式实验 store。
3. 不要保留全屏 mask，否则讲解浮窗仍会阻塞操作。
4. 收集率改为 5 秒结算后，前 5 秒显示为 0 是可接受行为，但 UI 文案要解释清楚。
5. Canvas key 一定要包含 `animationMode`，否则第 6 项无法彻底修复。
6. 伏安轨迹生命周期缩短后，测试中的时间推进需要整体缩短，否则测试会不稳定或等待过长。
7. `npm run lint` 和 `npm run format` 会写文件，执行前应确认本轮改动范围。
