     95 +## 5. 任务二：光子逸散到真空飞行区下边沿
     96 +

     98 +

    100 +
    101 +```text
    102 +cathode
    103 +anode
    104 +```
    105 +
    106 +当前比例大致为：
    107 +
    108 +```text
    109 +72% cathode
    110 +28% anode
    111 +```
    112 +
    113 +`updatePhoton()` 根据 `targetKind` 判断：
    114 +
    115 +- 命中阴极时生成 pulse，并可能生成电子。
    116 +- 命中阳极时可能触发阳极污染电子。
    117 +
    118 +### 5.2 计划改动
    119 +
    120 +新增第三类光子终点：
    121 +
    122 +```text
    123 +stray
    124 +```
    125 +
    126 +推荐概率分配：
    127 +
    128 +```text
    129 +5% stray
    130 +72% cathode
    131 +23% anode
    132 +```
    133 +
    134 +这样能够保留原有阴极命中强度，同时把部分原阳极方向光子转为逸散，减少对主光电效应视觉频率的扰动。
    135 +
    136 +`stray` 的终点规则：
    137 +
    138 +```text
    139 +targetX: random(surface, collectorX)
    140 +targetY: fieldBottom
    141 +```
    142 +
    143 +其中：
    144 +
    145 +- `surface` 是阴极右边界。
    146 +- `collectorX` 是阳极位置。
    147 +- `fieldBottom` 是真空飞行区下边沿。

    207 +const isVoltAmpMode = computed(() => principleMode.value === 'volt-amp')

    333 +## 9. 伏安特性模式的电子角度发射规划
    334 +
    335 +### 9.1 当前电子生成
    336 +
    337 +当前正常电子在 `spawnEvent()` 中大致使用：
    338 +
    339 +```js
    340 +vx = initialSpeed * resolveElectronInitialSpeedMultiplier()
    341 +vy = randomBetween(-36, 36)
    342 +```
    343 +
    344 +飞行开始后，`createElectronFlightState()` 接收 `initialSpeedPxPerS` 和 `initialVyPxPerS`，`stepElectronFlight()` 更新：
    345 +
    346 +```js
    347 +vx += acceleration * dt
    348 +x += vx * dt
    349 +y += vy * dt
    350 +vy *= 0.988
    351 +```
    352 +
    353 +这已经具备曲线运动基础，但当前 `vy` 只是小扰动，视觉上仍接近水平发射。
    354 +
    355 +### 9.2 计划改动
    356 +
    357 +伏安特性模式下，电子生成时以阴极右边界法线为基准随机偏转：
    358 +
    359 +```text
    360 +基准方向：+x
    361 +角度范围：-90 度到 +90 度
    362 +```
    363 +
    364 +实现建议：
    365 +
    366 +```js
    367 +const angle = randomBetween(-Math.PI / 2, Math.PI / 2)
    368 +const speed = initialSpeed * resolveElectronInitialSpeedMultiplier()
    369 +vx = speed * Math.cos(angle)
    370 +vy = speed * Math.sin(angle)
    371 +```
    372 +
    373 +为避免极端 `cos(±90°)` 导致电子几乎不进入真空区，可以做极小下限：
    374 +
    375 +```js
    376 +vx = Math.max(speed * 0.04, speed * Math.cos(angle))
    377 +```
    378 +
    379 +是否加入该下限取决于实际视觉效果。如果严格遵守 `-90° ~ +90°`，可以不加下限，只接受少量贴边上/下飞出。
    380 +
    381 +### 9.3 抛物线效果
    382 +
    383 +当前电场只影响 x 方向，因此伏安模式下电子轨迹会呈现：
    384 +
    385 +- `vy` 方向近似匀速。
    386 +- `vx` 方向受外加电压加速度影响。
    387 +- 在画面坐标中表现为弯曲曲线。
    388 +
    389 +当前 `stepElectronFlight()` 每帧有：
    390 +
    391 +```js
    392 +vy *= 0.988
    393 +```
    394 +
    395 +这会让纵向速度缓慢衰减，使轨迹不像严格抛物线。建议：
    396 +
    397 +- 截止电压模式保持现状。
    398 +- 伏安特性模式下关闭或弱化纵向阻尼。
    399 +
    400 +可选实现：
    401 +
    402 +```js
    403 +verticalDamping = animationMode === 'volt-amp' ? 1 : 0.988
    404 +```
    405 +
    406 +如果不想改 `physics.js` 函数签名，也可以在 `engine.js` 层用模式字段进行补偿，但更清晰的做法是扩展 `stepElectronFlight()` 参数。

    511 +## 14. 风险与处理建议
    512 +
    513 +### 14.1 正向高电压导致电子过快收集
    514 +
    515 +`+30V` 可能让电子快速冲到阳极，难以观察轨迹。
    516 +
    517 +处理建议：
    518 +
    519 +- 使用 `voltageReferenceMax = 30` 控制动画加速度归一化。
    520 +- 必要时给动画加速度设置视觉上限。
    521 +
    522 +### 14.2 伏安模式下文案残留
    523 +
    528 +- `PhotoelectricAnimationPanel.vue` 的统计、误差文案按 `animationMode` 分支。
    529 +- 伏安模式下只解释光电流、外加电压、电场力、收集率和伏安曲线。
    530 +
    534 +
    535 +处理建议：
    536 +
    537 +- 不改变电流公式，只改变视觉粒子路径。
    538 +- 概率从原阳极方向中扣除，尽量保持阴极命中比例。
    539 +
    540 +### 14.4 模式切换状态残留
    541 +
    542 +伏安模式下用户可能设置 `+30V`，再切回截止电压模式。
    543 +
    544 +处理建议：
    545 +
    546 +- 切换模式时主动夹取电压。
    547 +- 需要时让动画结构 reset key 包含 `animationMode`，避免旧模式粒子残留到新模式。
