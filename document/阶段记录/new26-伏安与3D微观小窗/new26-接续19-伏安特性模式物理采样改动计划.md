# 伏安特性模式物理采样改动 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 仅在伏安特性模式下引入以电子运动轨迹为依据的收集率模型，使默认实验参数下电压约 10 V 时收集率自然达到约 80%，同时保证截止电压模式行为不变。

**Architecture:** 保留截止电压模式现有经验映射路径，新建伏安特性模式专用物理采样路径。伏安模式的收集率由初速度、出射角、电场加速度、极板几何和边界窗口共同决定，不允许用电压条件直接强制收集率。动画引擎继续使用同源运动方程推进电子，曲线指标和粒子运动共享同一组伏安物理参数。

**Tech Stack:** Vue 3, Vite, Pinia, Vitest, Canvas 2D, project-local photoelectric animation physics modules.

---

## 1. 硬性边界

- 本计划只作用于 `animationMode === 'volt-amp'`。
- 不修改截止电压模式的默认初速度比例、加速度比例、收集率经验路径和截止状态判定。
- 若伏安模式与截止模式共用代码路径，必须先拆分为模式分派函数，再只改伏安分支。
- 不允许写入类似 `if (voltage === 10) return 0.8` 的收集率硬编码。
- 10 V 约 80% 是物理采样模型校准后的验收结果，不是直接赋值规则。
- 所有小阶段结束时必须生成阶段总结文档并提交 git。
- 由于依赖体积较大，默认不创建 worktree，继续使用当前 git 分支进行版本管理。

## 2. 物理依据

本地参考文档：

- `document/光电效应的伏安特性曲线.pdf`

该文档的关键结论：

- 光电子脱离阴极后，在两极板间真空中受加速电场作用，轨迹由初速度、方向角、板间电压和板距决定。
- 光电流可由电子到达阳极的空间分布函数积分得到。
- 正向加速电压增大时，电子在阳极的分布逐渐集中，光电流趋于饱和。
- 表 1 中 10 V 电流已经接近高电压平台区，适合作为伏安模式动画的校准验收点。

外部资料依据：

- OpenStax University Physics, Photoelectric Effect: `https://openstax.org/books/university-physics-volume-3/pages/6-2-photoelectric-effect`
- LibreTexts Photoelectric Effect: `https://phys.libretexts.org/Bookshelves/University_Physics/University_Physics_%28OpenStax%29/University_Physics_III_-_Optics_and_Modern_Physics_%28OpenStax%29/06%3A_Photons_and_Matter_Waves/6.03%3A_Photoelectric_Effect`

共同约束：

- 正向电压提高时光电流上升并趋于饱和。
- 反向电压达到截止电压时光电流趋近于零。
- 饱和电流主要由入射光强决定。
- 截止电压由光电子最大初动能决定。

## 3. 当前问题定位

### 3.1 正向电压收集率被强制设满

文件：`src/features/photoelectric-animation/mapping.js`

当前正向电压分支存在如下行为：

```js
} else if (voltage > VOLTAGE_NEUTRAL_EPSILON) {
  collectorSuccessRatio = 1
}
```

问题：

- 这会导致任何正向电压都近似满收集。
- 它绕过了电子初速度、出射角和加速度共同决定轨迹的物理过程。
- 伏安模式下 10 V 约 80% 无法通过物理运动自然产生。

### 3.2 初速度和加速度尺度没有伏安模式专用分离

文件：`src/features/photoelectric-animation/physics.js`

当前全局常量：

```js
export const EV_TO_ANIMATION_SPEED = 180
export const FIELD_TO_ANIMATION_ACCELERATION = 180
```

问题：

- 截止电压模式依赖这些比例，不应直接改动。
- 伏安模式要求电子初速度远小于电场加速度影响，需要独立尺度。
- 10 V 下应让 `0.5 * a * t^2` 在跨越极板过程中成为主导项，而不是让 `v0 * t` 主导。

### 3.3 粒子统计和理论收集率需要同源

文件：`src/features/photoelectric-animation/engine.js`

现状：

- 电子飞行已经通过 `stepElectronFlight()` 推进。
- 画布统计里有实际到达阳极的窗口统计。
- 但映射层给出的 `collectorSuccessRatio` 仍有经验设定和正压设满行为。

目标：

- 伏安模式的 `collectorSuccessRatio` 来自与动画同源的轨迹采样估计。
- 动画粒子的实际到达结果应与估计收集率保持一致趋势。

## 6. 总体验收标准

### 伏安特性模式

- 默认参数下，10 V 左右收集率约 80%，允许区间为 `75% - 85%`。
- 10 V 的收集率由轨迹采样得出，不由电压条件直接赋值。
- 30 V 收集率高于 10 V，并趋向饱和平台。
- 0 V 收集率低于 10 V。
- 负电压接近截止电压时收集率趋近 0。
- 电子运动表现为加速度影响显著，跨区运动中 `0.5 * a * t^2` 的贡献大于初速度横向贡献。
- 画布实时统计的收集率趋势与采样模型一致。

### 截止电压模式

- 现有截止电压模式行为保持不变。
- 正压、反压、截止附近、阈值以下等已有测试继续通过。
- 不读取伏安模式专用采样器。
- 不共享伏安模式专用速度和加速度尺度。

### 项目验证

- `npm run test:unit -- --run`
- `npm run build`
- 浏览器验收伏安模式 0 V、10 V、30 V、负向截止附近四个点。

## 7. 风险与处理

- 风险：当前 `mapping.js` 与 `engine.js` 中部分字段被多个模式共用。处理：先加截止模式回归测试，再拆分模式分支。
- 风险：采样器过重影响渲染。处理：采样器采用固定低样本数和纯函数计算，只在状态映射时运行。
- 风险：实时粒子统计存在随机波动。处理：测试使用确定性采样；浏览器验收允许实时窗口围绕目标值波动。
- 风险：PDF 论文模型含未知实验参数。处理：动画只吸收其轨迹和分布函数思想，参数以项目几何和教学可视化尺度校准，不伪装为真实仪器拟合值。
