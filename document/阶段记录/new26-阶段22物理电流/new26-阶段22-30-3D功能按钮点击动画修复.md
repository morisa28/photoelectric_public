# 阶段文档22-30-new(26)-3D功能按钮点击动画修复

## 1. 阶段目标

用户反馈：3D 实验台中不只是电压按钮，所有具备功能的按钮都失去了点击动画；旋钮不在本问题范围内。

本阶段目标：

1. 排查按钮功能链路与点击动画链路是否分离。
2. 修复功能按钮按下/释放时的视觉反馈。
3. 不改旋钮拖拽逻辑。
4. 不改底层物理计算、电压步进、模式切换等业务逻辑。

## 2. 排查结论

排查发现：

1. 业务链路仍然有效：
   - `Btn_Up` 仍映射到电压增加。
   - `Btn_Down` 仍映射到电压减少。
   - 直接调用按钮动作可以改变底层电压。

2. 点击动画链路缺失：
   - 当前自定义 3D 交互层会在捕获阶段拦截 pointer 事件。
   - 被识别为功能按钮后，事件会被 `preventDefault` 和 `stopPropagation` 拦截。
   - 因此 Spline 原生按钮动画无法收到原始 `pointerdown/pointerup`。
   - 代码中原本只把按钮记为诊断状态“按下中”，没有同步驱动 Spline 对象自身的按下/释放动画。

3. 旋钮不受影响：
   - 旋钮走拖拽控制路径。
   - 功能按钮走按钮点击路径。
   - 两条路径不同，因此出现“按钮无点击动画，但旋钮仍可交互”的现象。

## 3. 修改内容

本阶段在自定义按钮交互层中补入视觉反馈桥：

1. 按下功能按钮时：
   - 记录当前按钮为按下态。
   - 主动向 Spline 对象发出 `mouseDown` 事件。

2. 释放功能按钮时：
   - 先执行原有业务动作。
   - 延迟短时间后主动向 Spline 对象发出 `mouseUp` 事件。
   - 清除按下态。

3. 取消、离开、失焦或切换到拖拽时：
   - 立即发出释放事件。
   - 清除按下态，避免按钮卡在按下状态。

## 4. 影响范围

- 影响 3D 实验台功能按钮的点击动画反馈。
- 保留原有按钮业务映射：
  - 电压增减。
  - 电压精度增减。
  - 手动/自动切换。
  - 截止/伏安模式切换。
  - 电源、调零、接线、盖子等功能按钮。
- 不影响旋钮拖拽。
- 不影响光电流、截止电压、阳极寄生电流等物理计算。

## 5. 验证结果

已运行页面交互验证：

```text
页面：/3d-view?diagnostic=1&wait=2200&zoom=1.12
操作：真实鼠标按下 Btn_Up 热区，保持 80 ms 后释放
结果：按下期间诊断状态为“按下中”，释放后恢复“悬停”，电压从 -1.998 V 变为 -1.898 V。
页面错误：无
```

已运行构建：

```bash
npm run build
```

结果：

```text

```

## 6. 阶段结论

本问题不是近期电流曲线或物理框架改动直接破坏了按钮业务逻辑，而是 3D 自定义交互层截获 pointer 事件后，没有主动补回 Spline 原生按钮动画事件。

本阶段已将功能按钮的按下/释放动画显式接入自定义交互层，使按钮功能和按钮动画重新同步。
