## 5. 功能实现细节

### 5.1 整体实现方式

本项目采用“2D 主应用 + 3D iframe 页面 + 统一状态管理”的实现方式。

简单理解就是：

1. 2D 页面是主系统
2. 3D 页面嵌入在主系统中
3. 两边都不再各自保存一套实验数据
4. 而是统一使用一套实验状态

因此，2D 和 3D 的显示结果能够保持一致。

### 5.2 状态同步方式

2D 和 3D 之间通过消息通信进行双向同步。

同步逻辑是：

1. 2D 把当前实验状态发送给 3D
2. 3D 页面收到后更新自身界面
3. 用户若在 3D 中修改参数，3D 再把变更发回 2D
4. 2D 更新统一状态后，再同步给全部界面

这样可以保证：

1. 状态来源统一
2. 两边不会长期各自漂移
3. 数据记录结果一致

### 5.3 2D 与 3D 保持统一的关键点

本版本重点保证了三类统一：

1. 数据统一
   - 电压、频率、光强、材料、记录数据等保持一致
2. 器材统一
   - 盖子、连线、电源、距离、测试仪状态保持一致
3. 结果统一
   - 截止电压、光电流、波长、图表数据保持一致

### 5.4 3D 页面的技术实现说明

当前 3D 页面不是简单截图，也不是静态说明页，而是一个可交互的实验页面，包含：

1. 透视场景表现
2. 器材状态动画
3. 粒子动画
4. 图表展示
5. 实时数据显示
6. 控制面板

它的重点不只是“能看”，而是“能和 2D 同步工作”。
