# 阶段接续文档10-new(17)-实体线缆与光电管拖拽修正

## 本轮完成内容
- 已将原先基于屏幕投影的 `SVG` 导线替换为场景内实体圆线缆。
- 红线连接 `Cylinder_red_in -> Cylinder_red_out`，蓝线连接 `Cylinder_blue_in -> Cylinder_blue_out`。
- 线缆使用真实 3D 几何生成，路径为固定世界坐标，不会因相机转动而重新投影。
- 已修正 `Phototube_Box` 左键拖拽方向：鼠标右移时盒子右移，鼠标左移时盒子左移。

## 当前实现状态
- 线缆为场景内 `TubeGeometry + MeshStandardMaterial`。
- 线缆仅在场景初始化或 `photocellDistance` 变化时重建，不会随视角旋转改变坐标。
- 光电管盒子的 3D 位置同步系数已改为正向映射。
- 当前线缆路径按“桌面后缘固定走线”实现，已尽量避开起始主视角中的主要设备。

## 已验证结果
- `npm run build` 通过。
- 浏览器实测中，红蓝线缆都已生成。
- 浏览器实测中，`photocellDistance` 从 `40` 拖到 `49` 时，`Phototube_Box` 屏幕位置同步向右移动，方向正确。

## 后续建议
- 若后续需要继续优化观感，优先微调 `SplineScene.vue` 中的线缆控制点参数和材质参数。
- 若需要更贴近真实导线，可继续增加轻微下垂、端头护套或更明显的高光层次。
- 若后续发现某些特定视角仍有局部遮挡，优先调整固定走线走廊，不要回退到投影方案。

## 追加修正记录：后缘线缆可见性
- 保留红蓝实体线缆颜色，不改为黑灰线缆，增强线缆半径和自发光对比度。
- 保留“桌面后缘隐藏走线”方案，但将后缘通道从过深的遮挡物后方拉回接口后方的桌面后缘区域。
- 正面视角仍以局部露出为目标，避免通过强制置顶让线缆压在仪器、桌面或光电管上。
- 诊断模式追加线缆生成状态与后缘通道坐标，便于后续少量排查使用。
- Spline runtime 不稳定渲染运行时追加的普通 Three Mesh，当前保留 TubeGeometry 控制点，同时增加 canvas 可视层：正面只显示端点短段，偏后视角显示完整后缘走线。

## 追加修正记录：光电管拖动判定合并
- 旧实现中，光电管拖动命中同时支持 `raycast` 和 `projected-bounds` 两条判定路径。
- 当前已将光电管拖动统一为单一 `raycast` 判定入口，不再保留屏幕投影包围盒兜底。
- `pointermove` 悬停、`pointerdown` 开始拖拽、原生 `dragDrop` 拦截三处逻辑已统一复用同一个光电管命中结果，避免不同阶段命中标准不一致。
- 诊断面板中的“光电管命中”现在仅表示射线命中结果，便于后续继续排查真实命中链路。
