# new(8) 模块细分暂停点

## 1. 本轮目标

本轮没有继续修改 3D 主功能，也没有重做按钮、旋钮、镜头逻辑。

本轮只做一件事：按照《说明文档10-new(7)-3D界面修补开发指南》的结构要求，把 `new(8)` 当前 3D 相关职责进一步拆清楚，让后续开发不要继续把逻辑堆进一个场景文件。

## 2. 当前已完成的结构细分

### 2.1 主装配入口保留

- `src/three-lab/createPhotoelectricLabScene.js`

现在这个文件保留为“总装配入口”，负责：

1. 初始化 Three.js 基础对象
2. 加载 GLB / HDR
3. 组合各个子模块
4. 对外暴露 `applyState`、指针事件、键盘事件、释放接口

它不再继续承载所有视觉细节和节点整理细节。

### 2.2 新增细分模块目录

- `src/three-lab/photoelectric-lab/constants.js`
  - 3D 实验台专用常量
  - 模型与环境资源入口
  - 按钮交互 key 映射
  - 量程、滤光片、调零旋钮角度映射

- `src/three-lab/photoelectric-lab/objectUtils.js`
  - 节点交互挂载
  - 电压 / 电流显示格式化
  - 节点旋转轴处理
  - 模型局部中心 / 尺寸工具
  - 材质释放工具

- `src/three-lab/photoelectric-lab/bindModelNodes.js`
  - 统一绑定 GLB 里的关键节点
  - 集中维护模型命名入口
  - 在这里校验汞灯、光电管等关键节点是否缺失

- `src/three-lab/photoelectric-lab/registerLabInteractions.js`
  - 批量注册按钮、旋钮、滤光片转盘、接线柱等交互对象
  - 统一生成 `interactiveObjects` 与 `interactiveMap`
  - 集中处理按钮按压深度和材质克隆

- `src/three-lab/photoelectric-lab/createLabDynamicObjects.js`
  - 创建程序补充出来的光束、汞灯发光球、汞灯聚光灯
  - 保留线缆默认姿态

- `src/three-lab/photoelectric-lab/createLabScreenOverlays.js`
  - 左右显示屏 overlay 创建入口
  - 将屏幕显示和主场景装配解耦

- `src/three-lab/photoelectric-lab/createLabVisualState.js`
  - 统一收口视觉状态刷新
  - 包含：
    1. 汞灯与光束视觉
    2. 滤光片显示
    3. 线缆状态显示
    4. 旋钮角度同步
    5. 屏幕文本同步

- `src/three-lab/photoelectric-lab/createLabParticleSystem.js`
  - 光子 / 光电子粒子生成与回收
  - 运行态粒子更新

## 3. 这次拆分后，功能边界变得更明确

当前 3D 部分已经可以按下面的模块层级理解：

1. 页面层
   - `src/views/ThreeDView.vue`
   - `src/components/three-lab/PhotoelectricLabScene.vue`

2. 场景控制层
   - `src/composables/usePhotoelectricSceneController.js`

3. Three 场景装配层
   - `src/three-lab/createPhotoelectricLabScene.js`

4. 3D 实验台细分模块层
   - `src/three-lab/photoelectric-lab/*.js`

5. 场景状态归一化层
   - `src/three-lab/scene/snapshot.js`

6. 业务状态层
   - `src/stores/experiment.js`
   - `src/features/experiment/constants.js`
   - `src/features/experiment/helpers.js`

这样后面继续修 3D 时，可以先判断问题属于哪一层，再进入对应模块，不需要直接进总场景文件硬改。
