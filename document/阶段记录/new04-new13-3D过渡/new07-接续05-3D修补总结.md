# new(7) 3D修补接续总结

## 1. 本轮工作范围

本轮工作没有继续大规模重构 2D 页面，重点放在以下两件事：

1. 把 3D 页面切换到当前可用的完整本地模型与 HDR 资源。
2. 梳理现有项目结构、3D 入口和后续修补方案，并形成接续文档。

用户在本轮中明确要求：

1. 2D 与 3D 页面必须分开，不能同屏。
2. 2D 页面内点击按钮进入 3D 页面。
3. 3D 页面需要完整模型、正常光照、可交互按钮与旋钮。
4. 模型显示屏位置应覆盖两块小屏，显示真实实验数据。
5. 本轮最后需要输出接续总结文档，方便后续继续开发。

## 4. 当前仍存在的问题

下面这些问题当前**仍未彻底修完**，它们正是后续修补的重点。

### 4.1 场景光照仍偏暗

当前虽然接了 HDR、环境光和顶灯，但整体仍不够亮，尤其：

1. 房间四周偏暗
2. 实验台正面不够突出
3. 前方缺少补光

后续必须增加：

1. 正面补光
2. 侧后轮廓光
3. 更合理的曝光与环境强度

### 4.2 初始镜头仍未完全符合要求

当前镜头虽然已可看模型，但还没有完全达到以下目标：

1. 一打开 3D 页就正对实验台正面全貌
2. 试验台中心与当前视野中心严格重合
3. 镜头不能超出房间边缘

### 4.3 相机控制语义仍不完全正确

用户要求：

1. 鼠标滚轮缩放
2. 中键拖动旋转
3. `W/S/A/D` 分别对应平面方向上下左右移动
4. 旋转和平移都要平滑

当前缺口：

1. 还不是严格的“中键拖动旋转”
2. `W/S/A/D` 还不是完全独立的平面平移
3. 还没有按房间边界做实际约束

### 4.4 滤光片交互仍不符合最终要求

当前滤光片已有切换能力，但还不是用户要求的：

1. 鼠标左键按住并拖动滤光片切档
2. 共 5 档循环切换
3. 删除当前错误的点击旋转动画

这部分需要重做交互语义。

### 4.5 接线柱状态不够细

当前仍主要沿用一个总的 `cableConnected` 概念。

但用户要求是：

1. 汞灯处接线柱固定连接且不可改
2. 试验台后方接线柱初始断开
3. 断开时红光警示
4. 连上时绿光

因此后续必须拆分成至少两类状态：

1. 固定连接端
2. 可切换后端

### 4.6 电流量程档位数量不匹配

当前 `CURRENT_RANGES` 是 8 档。

用户要求是 6 档。

这部分一定要先改常量，再改旋钮角度映射。

### 4.7 旋钮旋转轴仍应进一步收口

用户要求所有可旋转物体都应围绕柱体高度方向为轴心旋转。

当前虽已支持拖动，但还没有把所有旋钮统一抽象成：

1. 旋转轴
2. 最小角
3. 最大角
4. 档位表 / 连续映射

后续应统一封装旋钮旋转控制。

### 4.8 汞灯发光语义仍需细化

目标要求：

1. 盖子未摘下时，柱体发红光
2. 盖子摘下时，柱体发白光并产生光柱
3. 电源关闭时，两种情况都不发光

当前基础表现已存在，但后续仍要严格按这三种状态重构。

## 7. 建议的后续结构拆分

当前 `src/three-lab/createPhotoelectricLabScene.js` 已经偏大，后续建议拆成：

```text
src/three-lab/
  createPhotoelectricLabScene.js
  createOrbitRig.js
  scene/
    loadLabModel.js
    bindLabNodes.js
    applyLighting.js
    registerInteractions.js
    applySceneState.js
    createScreenOverlay.js
    createInteractionAnimations.js
```

其中职责建议如下：

1. `loadLabModel.js`
   - 只负责加载 GLB / HDR

2. `bindLabNodes.js`
   - 只负责查找模型节点并校验

3. `applyLighting.js`
   - 只负责房间和实验台灯光

4. `registerInteractions.js`
   - 只负责 hover / click / drag 的语义绑定

5. `applySceneState.js`
   - 只负责根据 store 状态刷新模型表现

6. `createScreenOverlay.js`
   - 只负责屏幕贴片绘制

7. `createInteractionAnimations.js`
   - 只负责按钮与旋钮动画

## 9. 本轮结论

本轮已经完成了“把现有完整模型接到 3D 页面里，并把基础交互与模型显示屏跑通”的阶段目标。

但从用户最终要求看，这一版仍然属于：

**3D 页面主干已建立，后续需围绕相机控制、交互语义、线缆状态拆分、光照和动画细节继续修补。**

因此，下一轮不应再扩散新功能，而应严格按以下顺序修：

1. 相机
2. 线缆/接线柱状态
3. 电流量程 6 档
4. 滤光片拖动切档
5. 补光与细节动画
