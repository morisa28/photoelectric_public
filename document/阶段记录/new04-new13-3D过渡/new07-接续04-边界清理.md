## 继续接手时先记住三件事

### 1. 不要回退到“大文件全能”

如果后续继续改：

1. 页面控制不要直接塞回 `.vue` 脚本块
2. 常量不要重新塞回 `experiment.js`
3. 3D 对象搭建不要重新塞回 `createPhotoelectricLabScene.js`

### 2. 下阶段优先级应转入“验证 / 清理”或“交互清晰”

建议顺序：

1. 先验证和清理
2. 再做交互清晰
3. 最后才考虑视觉质感

不要反过来先做灯光、材质或性能优化。

### 3. 构建验证要避开项目内 dist

当前共享目录环境下，项目内 `dist/` 写入不稳定。

后续如果只是验证源码是否可编译，优先使用：

```bash
node node_modules/vite/bin/vite.js build --outDir /tmp/任意可写目录
```
