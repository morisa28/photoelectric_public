# 光电效应项目：截止电压模式电流计算逻辑修改建议与实施指导

## 1. 背景与目标

当前项目在截止电压模式下出现了明显偏离真实物理规律的现象：在 Cs 阴极、365/405/436/546/577 nm 滤光片条件下，程序测得的截止电压整体偏低。例如当前程序测得：

| 波长 | 当前程序记录电压 | 取绝对值 | 按项目 Cs 常量计算的理论截止电压 | 偏差 |
|---:|---:|---:|---:|---:|
| 365 nm | -1.297 V | 1.297 V | 1.447 V | -0.150 V |
| 405 nm | -1.009 V | 1.009 V | 1.114 V | -0.105 V |
| 436 nm | -0.813 V | 0.813 V | 0.895 V | -0.082 V |
| 546 nm | -0.297 V | 0.297 V | 0.320 V | -0.023 V |
| 577 nm | -0.185 V | 0.185 V | 0.199 V | -0.014 V |

这不是单个数据点的偶然误差，而是电流计算逻辑导致的系统性偏差。修改目标是：

1. 让阴极主光电流符合真实物理规律：反向电压达到截止电压后降为 0，而不是变成负值。
2. 将阴极光电流、阳极寄生电流、暗电流、本底电流、仪器零点偏置清晰分层。
3. 让截止电压记录逻辑基于统一的电流模型，而不是简单记录当前外加电压。
4. 保持项目教学可用性：既能展示正常测量，也能在显式开启异常演示时展示阳极污染、杂散光等导致的偏差。

---

## 2. 当前问题定位

### 2.1 阴极光电流公式不符合截止区物理规律

当前 `src/stores/experiment.js` 中的核心电流公式近似为：

```js
const current = saturationCurrent * Math.tanh((voltage + cutoff) / 0.45)
```

这会产生如下行为：

```text
U > -Uc      -> I > 0
U = -Uc      -> I = 0
U < -Uc      -> I < 0
```

但真实物理中，阴极发出的主光电流不应为负。反向电压超过截止电压后，阴极电子不能到达阳极，主阴极光电流应接近 0：

```text
U <= -Uc     -> I_cathode = 0
```

负电流只能来自阳极寄生光电流、暗电流、本底电流或仪器偏置，不能由阴极主光电流公式产生。

### 2.2 固定 `0.45 V` 过渡宽度过于粗糙

当前公式使用固定的 `0.45 V` 控制电流从正到负的过渡宽度。这个值没有随频率、最大初动能、电子角分布或光电管几何变化。

对 577 nm，Cs 理论截止电压只有约 0.199 V，`0.45 V` 已经大于截止电压本身，会导致低频端的截止区严重失真。对 365 nm，虽然理论截止电压约 1.447 V，但 `0.45 V` 仍会让截止附近的电流下降过于“软”，容易被零点偏置或阳极寄生电流提前抵消。

### 2.3 电流口径没有分层

当前主仪表读数大致为：

```js
currentPhotocurrentAmp = rearCableConnected
  ? generatedPhotocurrentAmp + referenceCurrentAmp
  : referenceCurrentAmp
```

这里把“光电流”和“仪器参考偏置”直接相加，且 `generatedPhotocurrentAmp` 本身又可能为负。这样会导致表头显示的零点并不等价于真实阴极光电流为零。

正确做法应当明确区分：

```text
I_cathode   阴极主光电流，非负
I_anode     阳极寄生光电流，通常为负或 0
I_dark      暗电流
I_bg        本底/背景电流
I_offset    仪器零点偏置/调零残差
I_meter     最终仪表读数
```

### 2.4 3D 微观小窗与主 store 的阳极电流口径不统一

目前 3D 微观小窗中已经存在 `parasiticAnodePhotocurrentAmp`，但主实验 store 的 `currentPhotocurrentAmp` 没有统一使用这一项。这会导致：

```text
微观动画看起来考虑了阳极污染，
但主电流表和记录数据并不一定使用同一套电流模型。
```

这会让 3D 表头、微观动画、实验记录三者在物理意义上不一致。

### 2.5 截止电压记录逻辑过于直接

当前记录截止电压的逻辑本质是：

```js
cutoffVoltage = Math.max(0, -currentAppliedVoltage)
```

也就是说，只要当前电压是 -1.297 V，就记录截止电压 1.297 V。程序并没有真正根据电流曲线求解 `I = 0` 的位置，也没有区分“表观零电流点”和“理论截止点”。

这会导致用户只要在一个被偏置项抵消后的表观零点按下记录，就会得到系统性偏低的截止电压。

## 3. 修改后的物理模型设计

建议将总电流拆成如下结构：

```text
I_total(U) = I_cathode(U) + I_anode(U) + I_dark + I_background
I_meter(U) = I_total(U) + I_offset
```

其中：

```text
I_cathode(U) >= 0
I_anode(U) <= 0
```

### 3.1 理论截止电压

继续使用光电效应方程：

```text
Uc = max(0, hν - W)
```

由于项目中 `PLANCK_EV` 的单位是 eV·s，且电压单位 V 对应 eV/e，代码中可继续写为：

```js
const cutoffVoltage = Math.max(0, PLANCK_EV * frequency - workFunctionEv)
```

### 3.2 阴极主光电流模型

建议用非负收集率模型替换 `tanh`：

```js
function clamp(value, min, max) {
  return Math.max(min, Math.min(max, value))
}

function cathodeCollectionRatio({ voltage, cutoffVoltage, intensityRatio }) {
  if (cutoffVoltage <= 0) {
    return 0
  }

  // 反向电压已经达到或超过截止电压：阴极主光电流为 0
  if (voltage <= -cutoffVoltage) {
    return 0
  }

  const zeroBiasCollection = clamp(
    0.58 + 0.22 * Math.sqrt(clamp(intensityRatio, 0, 1)),
    0.55,
    0.85,
  )

  // 遏止区：-Uc < U < 0
  if (voltage < 0) {
    const x = clamp((voltage + cutoffVoltage) / Math.max(cutoffVoltage, 0.001), 0, 1)
    return zeroBiasCollection * Math.pow(x, 1.25)
  }

  // 加速区：U >= 0，逐渐趋近饱和
  return zeroBiasCollection + (1 - zeroBiasCollection) * (1 - Math.exp(-voltage / 0.8))
}

function calculateCathodePhotocurrentAmp({
  voltage,
  cutoffVoltage,
  intensity,
  saturationCurrentMaxAmp,
  saturationCurrentMinAmp,
  noiseRatio = 0,
}) {
  const intensityRatio = clamp(Number(intensity) / 100, 0, 1)

  if (cutoffVoltage <= 0 || intensityRatio <= 0) {
    return 0
  }

  const saturationCurrent = Math.max(
    saturationCurrentMinAmp,
    intensityRatio * saturationCurrentMaxAmp,
  )

  const ratio = cathodeCollectionRatio({
    voltage: Number(voltage),
    cutoffVoltage,
    intensityRatio,
  })

  const current = saturationCurrent * ratio * (1 + Number(noiseRatio || 0))
  return Number(Math.max(0, current).toExponential(6))
}
```

这个模型满足：

```text
U <= -Uc        I_cathode = 0
-Uc < U < 0     I_cathode 随电压升高单调增大
U = 0           I_cathode 已有明显电流，但未必饱和
U > 0           I_cathode 逐渐趋近饱和
```

### 3.3 阳极寄生电流模型

阳极寄生电流应单独建模，且默认只占很小比例。建议模型：

```js
function wavelengthFactorForAnode({ photonEnergyEv, effectiveAnodeWorkFunctionEv }) {
  if (photonEnergyEv <= effectiveAnodeWorkFunctionEv) {
    return 0
  }

  const excess = photonEnergyEv - effectiveAnodeWorkFunctionEv
  return clamp(excess / 1.5, 0, 1)
}

function reverseVoltageCollectionFactor({ voltage, cutoffVoltage }) {
  if (voltage >= 0) {
    return 0
  }

  return clamp(
    Math.pow(Math.abs(voltage) / Math.max(cutoffVoltage, 0.08), 1.2),
    0,
    1,
  )
}

function calculateAnodeParasiticCurrentAmp({
  voltage,
  cutoffVoltage,
  frequency,
  intensity,
  anodeParasiticMaxAmp,
  anodeStrayLightRatio,
  anodeContaminationRatio,
  effectiveAnodeWorkFunctionEv,
}) {
  const photonEnergyEv = PLANCK_EV * Number(frequency)
  const intensityRatio = clamp(Number(intensity) / 100, 0, 1)
  const wavelengthFactor = wavelengthFactorForAnode({
    photonEnergyEv,
    effectiveAnodeWorkFunctionEv,
  })
  const voltageFactor = reverseVoltageCollectionFactor({
    voltage: Number(voltage),
    cutoffVoltage,
  })

  const magnitude = Number(anodeParasiticMaxAmp)
    * intensityRatio
    * clamp(anodeStrayLightRatio, 0, 1)
    * clamp(anodeContaminationRatio, 0, 1)
    * wavelengthFactor
    * voltageFactor

  return Number((-magnitude).toExponential(6))
}
```

推荐默认参数：

```js
export const ANODE_PARASITIC_CURRENT_MAX_AMP = 2e-13
export const ANODE_STRAY_LIGHT_RATIO_DEFAULT = 0.08
export const ANODE_CONTAMINATION_RATIO_DEFAULT = 0.25
export const EFFECTIVE_ANODE_WORK_FUNCTION_EV_DEFAULT = 2.6
```

这些默认值会让正常阳极寄生电流保持在小负值范围内，不会轻易把 365 nm 的截止电压提前 0.15 V。若需要演示严重阳极污染，可另外设置“异常演示模式”，不要把严重污染作为默认真实实验状态。

cathodePhotocurrentAmp() {

    voltage: this.params.voltage,

anodeParasiticCurrentAmp() {
  if (!this.isInstrumentPowered || !this.photoelectricConditionsReady) {
    return 0
  }

  return calculateAnodeParasiticCurrentAmp({
    voltage: this.params.voltage,
    cutoffVoltage: this.currentCutoffVoltage,
    frequency: this.params.frequency,
    intensity: this.params.intensity,
    anodeParasiticMaxAmp: ANODE_PARASITIC_CURRENT_MAX_AMP,
    anodeStrayLightRatio: ANODE_STRAY_LIGHT_RATIO_DEFAULT,
    anodeContaminationRatio: ANODE_CONTAMINATION_RATIO_DEFAULT,
    effectiveAnodeWorkFunctionEv: EFFECTIVE_ANODE_WORK_FUNCTION_EV_DEFAULT,
  })

totalPhotoelectricCurrentAmp() {
  return Number((
    this.cathodePhotocurrentAmp
    + this.anodeParasiticCurrentAmp
    + this.darkCurrentAmp
    + this.backgroundCurrentAmp

currentPhotocurrentAmp() {
  if (!this.isInstrumentPowered) {
    return 0

  return this.apparatus.rearCableConnected
    ? Number((this.totalPhotoelectricCurrentAmp + this.referenceCurrentAmp).toExponential(6))
    : this.referenceCurrentAmp
}

### 步骤 6：实现 apparent cutoff 求解

可使用二分法在 `[-2, 0]` 区间寻找总光电流零点：

```js
export function solveApparentCutoffVoltage({
  cutoffVoltage,
  currentAtVoltage,
  minVoltage = -2,
  maxVoltage = 0,
  toleranceCurrentAmp = 0.05e-13,
  toleranceVoltage = 0.001,
}) {
  let left = minVoltage
  let right = maxVoltage
  let fLeft = currentAtVoltage(left)
  let fRight = currentAtVoltage(right)

  if (!Number.isFinite(fLeft) || !Number.isFinite(fRight)) {
    return null
  }

  // 如果区间没有跨过 0，不能硬算
  if (fLeft * fRight > 0) {
    return null
  }

  for (let i = 0; i < 40; i += 1) {
    const mid = (left + right) / 2
    const fMid = currentAtVoltage(mid)

    if (Math.abs(fMid) <= toleranceCurrentAmp || Math.abs(right - left) <= toleranceVoltage) {
      return Number(Math.abs(mid).toFixed(4))
    }

    if (fLeft * fMid <= 0) {
      right = mid
      fRight = fMid
    } else {
      left = mid
      fLeft = fMid
    }
  }

  return Number(Math.abs((left + right) / 2).toFixed(4))
}
```

注意：如果阳极寄生电流存在，`apparentCutoffVoltage` 会略小于 `theoreticalCutoffVoltage`。但在默认正常参数下，这个偏差应很小。若偏差达到 0.1 V 以上，应提示“阳极污染/杂散光过强，测量结果不可靠”。

## 5. 参数建议

### 5.1 饱和电流范围

当前项目最大饱和电流为：

```js
PHOTOCURRENT_SATURATION_CURRENT_MAX_AMP = 20e-13
```

这个量级可暂时保留，因为在 `1e-13 A` 档下最大显示约 20，足够可见。但需要避免在截止点附近通过错误公式产生过大的残余正电流。

如需让伏安特性区更明显，可调到：

```js
PHOTOCURRENT_SATURATION_CURRENT_MAX_AMP = 40e-13
```

不建议一开始直接调到过大，否则 `1e-13 A` 档会频繁溢出。

### 5.2 阳极寄生电流默认范围

正常教学仿真中，在 Cs、365 nm、`U = -1.998 V`、无暗电流和本底电流的条件下：

```text
I_cathode = 0
I_anode   应为小负值
```

推荐目标显示范围：

```text
1e-13 A 档：-0.3 ～ 0.0
```

保守上限：

```text
1e-13 A 档：-1.0 ～ 0.0
```

不应出现：

```text
+1.6    代表截止后仍有明显正向阴极光电流
-16     代表严重异常阳极污染或符号/缩放错误
```

因此正常默认阳极寄生电流建议控制在：

```text
0 ～ -3e-14 A
```

最多不超过：

```text
-1e-13 A
```

## 6. 对当前五组截止电压数据的修正预期

在 Cs 阴极、无暗电流、无本底电流、正常阳极寄生电流很小的情况下，记录值应接近：

| 波长 | 建议理论目标 | 正常阳极寄生下可接受范围 |
|---:|---:|---:|
| 365 nm | 1.447 V | 1.42 ～ 1.45 V |
| 405 nm | 1.114 V | 1.09 ～ 1.12 V |
| 436 nm | 0.895 V | 0.87 ～ 0.90 V |
| 546 nm | 0.320 V | 0.30 ～ 0.33 V |
| 577 nm | 0.199 V | 0.18 ～ 0.21 V |

如果项目目标是拟合真实教学实验数据，而不是纯 Cs 理想模型，还需要额外引入：

```text
W_eff：有效逸出功
U_contact：接触电势修正
U_anode_error：阳极污染导致的系统偏差
```

但这些应作为显式参数，而不是混入错误的 `tanh` 主电流公式中。

## 11. 最终判断

当前程序数据偏低的根本原因是：

```text
主阴极光电流被错误建模为可正可负的 tanh 曲线；
截止区电流下降宽度固定且过宽；
阳极寄生电流、调零偏置和主电流没有清晰分层；
记录逻辑又直接把当前外加电压当作截止电压。
```

因此修复重点不应只是调大或调小某个常量，而应重构电流计算链路。只要将阴极主光电流、阳极寄生电流、仪器偏置、记录逻辑分离，当前这类系统性偏低问题就可以稳定消除。
