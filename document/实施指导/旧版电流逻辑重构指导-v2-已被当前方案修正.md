# 光电效应项目当前版本：电流计算逻辑重构修改指导

> 适用范围：基于当前远端 `main` 版本，对截止电压模式与伏安特性模式共用的电流计算逻辑进行重构。本文重点指导 Codex 修改 `src/stores/experiment.js`、`src/features/experiment/constants.js`、3D 微观小窗电流口径，以及相关单元测试。

---

## 1. 当前问题结论

当前版本的核心问题不是 3D 仪表贴图，也不是单纯记录表显示错误，而是 **主电流模型本身不符合光电效应真实物理规律**。

当前 `generatedPhotocurrentAmp()` 中使用：

```js
const current = saturationCurrent * Math.tanh((voltage + cutoff) / 0.45)
```

这会造成：

```text
U > -Uc      阴极光电流为正
U = -Uc      阴极光电流为 0
U < -Uc      阴极光电流变成负值
```

但真实物理中，阴极主光电流不应变成负值。反向电压超过截止电压后，阴极发出的电子不能到达阳极，阴极光电流应接近 0；负电流只能来自阳极寄生光电流、暗电流、本底电流或仪器偏置。

因此必须把总电流拆分成物理分层模型，而不是继续用一条 `tanh` 曲线代表所有现象。

## 3. 新电流模型设计原则

总电流必须拆成以下分量：

```text
I_total_display(U) = I_cathode(U) + I_anode(U) + I_dark + I_background + I_reference
```

在当前项目阶段，可以先实现：

```text
I_physical(U) = I_cathode(U) + I_anode(U)
I_meter(U) = I_physical(U) + I_reference
```

各分量物理约束如下：

### 3.1 阴极主光电流 `I_cathode`

必须满足：

```text
I_cathode >= 0
U <= -Uc 时，I_cathode = 0
-Uc < U < 0 时，I_cathode 随 U 增大而单调增加
U >= 0 时，I_cathode 逐渐趋近饱和电流 Is
```

不要再允许阴极主光电流为负。

### 3.2 阳极寄生电流 `I_anode`

阳极寄生电流用于描述阳极污染、杂散光照射阳极导致的反向光电流。

它应满足：

```text
I_anode <= 0
无光照时 I_anode = 0
电压 U >= 0 时 I_anode 接近 0
反向电压越大，I_anode 绝对值越大，随后趋于饱和
365 nm 下最明显，长波长下减小或接近 0
```

阳极寄生电流必须作为单独分量进入主 store，而不是只存在于 3D 微观动画组件中。

### 3.3 仪器零点/偏置 `I_reference`

`referenceCurrentAmp` 是仪器偏置和调零旋钮造成的基准电流，只应用于仪表显示与调零流程，不应被误当成物理光电流。

建议分清：

```text
cathodePhotocurrentAmp       阴极主光电流
anodeParasiticCurrentAmp     阳极寄生光电流
physicalPhotocurrentAmp      二者合计
meterCurrentAmp              仪表最终读数，即 physicalPhotocurrentAmp + referenceCurrentAmp
```

## 4. 建议新增文件：`src/features/experiment/currentModel.js`

为了避免继续把物理模型写散在 store 和组件中，建议新建：


用于集中管理电流模型。

### 4.1 推荐实现

```js
export function clamp(value, min, max) {
  return Math.max(min, Math.min(max, value))
}

export function calculateCathodeSaturationCurrentAmp({
  intensity,
  minAmp,
  maxAmp,
}) {
  const normalizedIntensity = clamp(Number(intensity || 0) / 100, 0, 1)
  if (normalizedIntensity <= 0) {
    return 0
  }
  return Math.max(minAmp, normalizedIntensity * maxAmp)
}

export function calculateCathodeCollectionRatio({
  voltage,
  cutoffVoltage,
  intensity,
}) {
  const numericVoltage = Number(voltage || 0)
  const safeCutoff = Math.max(0, Number(cutoffVoltage || 0))
  const normalizedIntensity = clamp(Number(intensity || 0) / 100, 0, 1)

  if (safeCutoff <= 0 || normalizedIntensity <= 0) {
    return 0
  }

  // 超过截止电压：阴极电子不能到达阳极，主阴极光电流为 0。
  if (numericVoltage <= -safeCutoff) {
    return 0
  }

  // 零偏压下非包围式光电管并非必然达到饱和，给一个合理收集率。
  const zeroBiasCollection = clamp(
    0.58 + 0.22 * Math.sqrt(normalizedIntensity),
    0.55,
    0.84,
  )

  // 反向遏止区：从 0 平滑增长到零偏压收集率。
  if (numericVoltage < 0) {
    const x = clamp((numericVoltage + safeCutoff) / safeCutoff, 0, 1)
    return zeroBiasCollection * Math.pow(x, 1.25)
  }

  // 正向加速区：逐渐接近饱和。
  return zeroBiasCollection + (1 - zeroBiasCollection) * (1 - Math.exp(-numericVoltage / 0.8))
}

export function calculateCathodePhotocurrentAmp({
  voltage,
  cutoffVoltage,
  intensity,
  minSaturationAmp,
  maxSaturationAmp,
  noiseRatio = 0,
}) {
  const saturationCurrent = calculateCathodeSaturationCurrentAmp({
    intensity,
    minAmp: minSaturationAmp,
    maxAmp: maxSaturationAmp,
  })
  const collectionRatio = calculateCathodeCollectionRatio({
    voltage,
    cutoffVoltage,
    intensity,
  })
  const current = saturationCurrent * collectionRatio * (1 + Number(noiseRatio || 0))
  return Number(Math.max(0, current).toExponential(6))
}
```

这套模型虽然仍是教学仿真近似，但至少满足关键物理约束：

```text
阴极主光电流永不为负；截止电压后归零；正向电压下趋于饱和。
```

## 5. 阳极寄生电流统一模型

建议从 `ThreeDMicroAnimationPanel.vue` 中迁移阳极寄生电流逻辑到 `currentModel.js`。

### 5.1 推荐新增常量

在 `src/features/experiment/constants.js` 中新增：

```js
export const ANODE_PARASITIC_CURRENT_MAX_AMP = 3e-13
export const ANODE_CONTAMINATION_RATIO = 0.18
export const ANODE_STRAY_LIGHT_RATIO = 0.16
export const ANODE_PARASITIC_WORK_FUNCTION_EV = 2.35
```

说明：

- 当前 3D 小窗中 `7e-13 * 0.52 * 0.58` 的组合偏强，容易让阳极寄生电流接近或超过正常截止测量可接受范围。
- 默认正常实验应使用较小的阳极寄生电流；严重污染可以另做演示模式，而不应作为真实测量默认状态。
- 对 `1e-13 A` 量程，正常阳极寄生电流建议控制在 `-0.3 ~ 0.0` 读数范围，极端保守不应低于 `-1.0`。

### 5.2 推荐函数

```js
export function calculateAnodeParasiticCurrentAmp({
  voltage,
  intensity,
  photonEnergyEv,
  anodeWorkFunctionEv,
  maxAmp,
  contaminationRatio,
  strayLightRatio,
}) {
  const numericVoltage = Number(voltage || 0)
  const normalizedIntensity = clamp(Number(intensity || 0) / 100, 0, 1)
  const excessEnergy = Math.max(0, Number(photonEnergyEv || 0) - Number(anodeWorkFunctionEv || 0))

  if (
    numericVoltage >= -0.001
    || normalizedIntensity <= 0
    || excessEnergy <= 0
  ) {
    return 0
  }

  const energyFactor = clamp(excessEnergy / 1.4, 0, 1)
  const reverseVoltageRatio = clamp(Math.abs(numericVoltage) / 2, 0, 1)
  const voltageFactor = reverseVoltageRatio ** 1.25

  const current = -Math.abs(maxAmp)
    * clamp(contaminationRatio, 0, 1)
    * clamp(strayLightRatio, 0, 1)
    * normalizedIntensity
    * energyFactor
    * voltageFactor

  return Number(current.toExponential(6))
}
```

## 8. 截止电压记录逻辑修改建议

当前 `measureCutoffVoltageFromAppliedVoltage()` 只是取当前电压绝对值：

```js
Math.max(0, -numericVoltage)
```

这可以保留为 `appliedStoppingVoltage`，但不应再直接称为真实测得截止电压。

### 8.1 推荐新增字段

记录数据时同时保存：

```js
appliedStoppingVoltage: Math.max(0, -this.params.voltage),
physicalCurrentAtRecordAmp: this.physicalPhotocurrentAmp,
meterCurrentAtRecordAmp: this.currentPhotocurrentAmp,
theoreticalCutoffVoltage: this.currentCutoffVoltage,
cutoffVoltage: measuredOrApparentCutoffVoltage,
```

### 8.2 最小改动方案

若暂时不实现自动求根，可以先把手动记录条件改严：

```text
只有当 physicalPhotocurrentAmp 的绝对值小于阈值时，才允许记录 cutoffVoltage。
```

不要只比较 `currentPhotocurrentAmp` 和 `zeroReferenceCurrent`，因为这会把仪器偏置也当成物理零点。

建议阈值：

```js
const CUTOFF_PHYSICAL_CURRENT_TOLERANCE_AMP = 0.3e-13
```

在 cutoff 模式下：

```js
if (Math.abs(this.physicalPhotocurrentAmp) > CUTOFF_PHYSICAL_CURRENT_TOLERANCE_AMP) {
  return '当前物理光电流尚未接近截止，请继续调节反向电压。'
}
```

### 8.3 更推荐方案：求解表观截止电压

在 `currentModel.js` 中新增求根函数：

```js
export function solveApparentCutoffVoltage({
  minVoltage = -2,
  maxVoltage = 0,
  currentAtVoltage,
  toleranceAmp = 0.1e-13,
  maxIterations = 48,
}) {
  let left = minVoltage
  let right = maxVoltage

  for (let index = 0; index < maxIterations; index += 1) {
    const mid = (left + right) / 2
    const current = currentAtVoltage(mid)

    if (Math.abs(current) <= toleranceAmp) {
      return Number(Math.abs(mid).toFixed(4))
    }

    // 约定：left 更偏负，right 更接近 0。
    // 如果 mid 处电流为正，说明反向电压还不够，应往更负方向移动。
    if (current > 0) {
      right = mid
    } else {
      left = mid
    }
  }

  return Number(Math.abs((left + right) / 2).toFixed(4))
}
```

注意：如果引入阳极寄生电流，`I_total = 0` 的表观截止点可能略小于理论截止电压，这是正常系统误差；但不应偏离过大。
